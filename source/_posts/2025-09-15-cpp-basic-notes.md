## g++ 基本命令和选项

基本用法：

```bash
g++ [options] source_files -o output_file
```

- `options`：编译器的选项参数
- `source_files`：要编译的源代码文件
- `-o output_file`：生成的可执行文件的名称

### 可选项

- `-c` 只进行编译，生成.o 文件，不进行链接操作
- `-E` 只进行预处理，生成预处理后的.i 文件
- `-S` 只编译到汇编代码，生成汇编文件 (.s)
- `-save-temps` 保存中间文件 (预处理、汇编、目标文件)
- `-std=c++XX` 指定要使用的 C++ 标准版本，如 `-std=c++11`、`-std=c++14` 等
- `-I dir` 指定头文件目录
- `-L dir` 指定链接库目录
- `-l library` 指定要链接的库文件
- `-O level` 指定优化级别，常见的有 `-O0`（无优化）、`-O1`、`-O2`、`-O3`（最高级别的优化），以及 `-Os`（针对空间优化）等
- `-g` 生成调试信息
- `-Wall` 启用所有警告信息
- `-Werror` 将警告视为错误

## C++ 是如何工作的

**编译器**：为每个.cpp 文件生成.obj(linux 下是.o) 文件

**链接器**：把所有.obj 文件合并成一个.exe 文件

**声明**：只给出函数的名称，参数，返回值，如：

```c++
void log(const char* str); // 在 main.cpp 文件中
```

这是编译器会相信有这个函数的定义从而不会报错，可以在另一个文件中实现这个函数，也就是函数的定义：

```c++
#include<iostream>
void log(const char* str)
{
    std::cout << str << std::endl;
}
```

## C++ 编译器是如何工作的

首先处理预处理语句 (所有预处理语句都以 `#` 开头)，然后把所有的代码转换成常量的数据或指令

每一个.cpp 文件都会生成一个.o 文件

在编译阶段报错的错误代码以 C 开头，如语法错误

## C++ 链接器是如何工作的

必须有 main 函数 (入口点，不一定叫 main，默认是 main)

在链接阶段报错的错误代码以 LNK 开头，如无 main 函数，找不到已被声明函数的定义

在函数名之前加上 `static` 关键字表明这个函数只作用在当前文件中：

```c++
#include <iostream>

void log(const char* str);

static int Multiply(int a,int b)
{
    log("Multiply");
    return a * b;
}
```

即便 `Multiply` 没有被调用，如果没有找到 `log`，链接器也会报错。但是在 `Multiply` 前面加上 `static` 就不会报错。

> [!NOTE]
>
> 函数的定义与声明的函数名、返回类型、参数必须完全相同才不会链接错误

在头文件里只放函数的声明也是为了防止函数的重复定义

## 函数

主要目的是防止代码重复

指定了返回值的函数必须有返回值，除了 `main` 函数。

把函数声明写在 `.h` 文件，函数的定义写在 `.cpp` 文件

## 头文件

`#include` 的作用就是复制粘贴

头文件通常用于声明某种类型的函数，声明是告诉编译器函数是存在的，只是在别的地方定义

c++ 头文件的开头会有 `#pragma once`，本质上是发送到编译器的预处理指令，为了防止单个头文件被同一个 `.cpp` 文件多次包含，尤其是为了防止头文件中的结构体重复定义

另一种防止重复包含的方法：

```c++
#ifndef _LOG_H
#define _LOG_H
...
#endif
```

即检查 `_LOG_H` 是否被定义了，如果没有定义就在编译中包含下面的代码，如果已经被定义了就都不会包含

可以在 include 的时候指定头文件路径，如：

```c++
#include "../include/log.h"
```

C 标准库一般有 `.h` 扩展名，C++ 标准库一般没有

## 指针

指针只是存储地址的整数

C++ 动态内存分配：

```c++
// 分配 8 个字节的内存，buffer 是指向那块内存开始的指针
char* buffer = new char[8];
// 用给定的数据填充内存块
memset(buffer,0,8);
delete[] buffer; // 回收这块内存
```

用 `new` 动态分配的内存空间，一定要用 `delete` 运算符进行释放

## 引用

是指针的伪装，更易理解，但是必须引用已经存在的变量，不能为空，且并不是新的变量，并不占用内存，没有真正的存储空间，某个变量的引用，等价于这个变量，相当于该变量的一个别名。

> [!NOTE]
>
> - 定义引用时一定要将其初始化成引用某个变量
> - 初始化后，它就一直引用该变量，不会再引用别的变量了
> - 引用只能引用变量，不能引用常量和表达式

```c++
int n = 4;
int & r = n;  // r 引用了 n, r 的类型是 int&
```

比较好的用法——用引用简化指针操作：

```c++
// 用指针实现
void Increment(int* value)
{
    (*value)++;
}

// 用引用实现：更简洁
void Increment(int& value)
{
    value++;
}
```

## 类

面向对象的程序设计方法：将某类客观事物共同特点（属性）归纳出来，形成一个数据结构（可以用多个变量描述事物的属性）；将这类事物所能进行的行为也归纳出来，形成一个个函数，这些函数可以用来操作数据结构 (这一步叫"抽象"）。然后，通过某种语法形式，将数据结构和操作该数据结构的函数"捆绑"在一起，形成一个"类"，从而使得数据结构和操作该数据结构的算法呈现出显而易见的紧密关系，这就是**"封装"**。

面向对象的程序设计具有"抽象"、"封装"、"继承"、"多态"四个基本特点。

实际上，"类"看上去就像"带函数的结构"。类内的函数称为方法。

```c++
class Rectangle
{
public:
    int w,h;
    int Area() // 算面积
    {
        return w * h;
    }
    int Perimeter() // 算周长
    {
        return 2 * (w + h);
    }
    void Init(int w_,int h_)
    {
        w = w_;  h = h_;
    }
}; // 注意这里的分号
```

## Static 关键字

### 在类和结构体外部使用 static

被声明为 static 的符号只对当前文件可见（只会在当前文件内部链接）

主要是为了防止使用全局变量

### 在类和结构体内部使用 static

- 普通成员变量每个对象有各自的一份，而静态成员变量一共就一份，为所有对象共享
- 普通成员函数必须具体作用于某个对象，而静态成员函数并不具体作用于某个对象
- 因此静态成员不需要通过对象就能访问
- 静态成员变量本质上是类内的全局变量，哪怕一个对象都不存在，类的静态成员变量也存在
- 静态成员函数本质上是类内的全局函数

### 在局部作用域使用 static

变量的生存期：在被删除之前，变量实际存在于内存的时间

变量的作用域：我们可以访问变量的范围

局部静态：生存期和整个程序一样，作用范围被限制在某个作用域（如函数）

```c++
void Function()
{
    static int i;
    i++;
    std::cout << i << std::endl;
}
```

## 枚举

枚举是一种命名值的方法，主要用于使用整数来表示某些状态或者数值时，默认从 0 开始

```c++
enum Example
{
    A, B, C // 默认 A=0，B=1，C=2
};

enum Example : unsigned char // 类型只能是整数的
{
    A=5, B, C // A=5，B=6，C=7
};
```

## 构造函数

构造函数是一种特殊的函数，每次构造一个对象的时候都会调用，名字与类名相同，可以有参数，不能有返回值，作用是对对象进行初始化，如给成员变量赋初值。如果定义类时没写构造函数，则编译器生成一个默认的无参数的构造函数。

## 析构函数

在销毁对象时被调用，没有参数和返回值，一个类最多只能有一个析构函数。析构函数对象消亡时即自动被调用。可以定义析构函数来在对象消亡前做善后工作，比如释放分配的空间等。

名称是 `~<类名>`

## 继承

在定义一个新的类 B 时，如果该类与某个已有的类 A 相似 (指的是 B 拥有 A 的全部特点)，那么就可以把 A 作为一个基类，而把 B 作为基类的一个派生类 (也称子类)。

派生类拥有基类的全部成员函数和成员变量。

写法：

```c++
class 派生类名：public 基类名
{
    // ...
};
```

## 虚函数

基类里面的虚函数在子类里面可以修改成别的功能

```c++
class Entity
{
public:
    virtual std::string GetName(){return "Entity:";}
};

class Player:public Entity
{
private:
    std::string m_Name;
public:
    Player(const std::string& name):m_Name(name) {}
    std::string GetName() override {return m_Name;}
};
```

## 面向对象

对象就是东西，以变量形式出现

对象 = 属性 (数据)+ 服务 (函数)

只能通过提供的服务来操作数据

**面向对象编程的原则**：

- 所有东西都是对象
- 程序运行的时候，对象互相告诉对方去做什么（而不是如何去做）
- 每个对象有自己的内存，自己的内存由其他对象组成
- 每一个对象都有一个类型
- 所有可以接受相同消息的对象可归为一类
- 对象以接口和外界打交道

Class（类）是概念，Objects（对象）是实体

类定义了对象长什么样，对象是类的实例
