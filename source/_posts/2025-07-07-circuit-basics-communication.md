# 电路基础与通信协议详解

## 一、电路结构和电路调试

### 1.1 常见缩写

| 缩写 | 全称 | 说明 |
|------|------|------|
| GND | Ground | 地，整个电路的参考点，决定了电压基准 |
| VCC | Circuit | 接入板子正向电源的电压 |
| VDD | Device | 集成 IC 的电压 |
| VEE | Emitter | 在电路指代电源负极 |
| VSS | Series | 公共连接点，约等于 GND |
| Vpp | Peak | 电压峰峰值，最大值 - 最小值 |
| VBAT | Battery | 电源电压 |

> [!NOTE]
> 直流电源分为单电源和双电源：
> - **单电源**：电压只有高于 GND 的部分
> - **双电源**：既有正电压又有负电压

### 1.2 电路结构

#### IO 口连接方式

##### 杜邦线与排针

<img src="https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefined%E6%9D%9C%E9%82%A6%E7%BA%BF.jpg" alt="杜邦线" style="zoom:150%;" />
_杜邦线与排针_

- 简单但是连接不可靠，且传输大电流发烫严重、损耗大
- 只适用于简单的东西

##### SMA(同轴线缆)

![SMA 连接器](https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefinedSMA.jpg)
_SMA 同轴线缆_

- 用于高频或对信号要求比较高的场景

##### FPC 连接器

![FPC 连接器](https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefinedFPC.jpg)
_FPC 连接器_

- 用于两块板子的连接，集成度高、简单、信号稳定性比较好

#### 供电管理

> [!NOTE]
> 通常板子的供电和芯片所需电压不一样

##### LDO(低压差线性稳压器)

LDO 是低压差线性稳压器

- **优势**：伺服电路结构简单、输出电压较稳定
- **劣势**：效率低，发热量大（压差损耗功率全部在 LDO 芯片上转化为热量），能够承载的功率低，一般只做降压电路

##### DC-DC 转换器

- **原理**：通过快速开启和关闭开关元件（MOSFET 晶体管）输出 PWM 波来调节电压或电流的平均值
- **优势**：效率高，能承载较大电流
- **劣势**：
  1. 电路结构复杂
  2. 有开关高频信号，干扰信号传输

> [!TIP]
> 电流较大一般用 DC-DC

### 1.3 电路调试

#### 常用工具

1. **万用表**：蜂鸣档在短路的时候会响
2. **信号发生器**：产生测试信号
3. **学生电源**：第一次上电把限流调的低一些，防止由于短路烧毁
4. **示波器**：调试之前先确认电气连接

#### 调试步骤

1. 先检查是否短路，蜂鸣档测 VCC 和 GND，显示 OL 说明没短路
2. 测量时保证不要同时碰到芯片的两个角
3. 触发电平在波形范围内才能让波形稳定显示

## 二、GPIO详解

**GPIO** = General Purpose Input Output（通用输入输出）

### 2.1 输出模式

#### 推挽输出

两个 MOS 管：强上拉强下拉，在高低两种电平下都具有驱动 (输出电流) 能力。但是驱动能力受芯片输出能力的限制

![推挽输出](https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefined%E6%8E%A8%E6%8C%BD%E8%BE%93%E5%87%BA.png)
_推挽输出电路_

#### 开漏输出

一个 MOS 管：强下拉，外部需要接上拉电阻，高电平输出能力由 VCC 提供，输出能力强

![开漏输出](https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefined%E5%BC%80%E6%BC%8F%E8%BE%93%E5%87%BA.png)
_开漏输出电路_

> [!TIP]
> VCC 是天花板，GND 是地板，推挽输出看做一只手把物块推到上面或者拉到下面
> 上拉电阻类似弹簧，不拉就会收回去

### 2.2 输入模式

#### 上拉输入

输入接上拉电阻，悬空时输入高电平，防止悬空时电压不可控（有效信号是低电平，常态高电平）

#### 下拉输入

与上拉输入相反（有效信号是高电平，常态低电平）

> 例如按键有效电平是低电平，应该采用上拉输入

## 三、USB 和电源简介

### 3.1 USB 接口

**USB** = Universal Serial Bus（通用串行总线）

- **串行**：所有信号从一根线传输按照时间先后顺序传输数据，稳定性比并行的强一些
- **总线**：所有设备挂在总线上，可以减少对引脚资源的占用

#### USB Type-A(2.0)

四根线：
- VCC
- GND
- DP(数据正线)
- DN(数据负线)

差模信号的电压取决于 DP 和 DN 之间的电压（差分传输），防止环境噪声干扰。

> [!NOTE]
> 差分走线必须等长、等宽、紧密靠近、在同一层面

#### USB Type-C

![USB Type-C](https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefinedtype-C.jpg)
_USB Type-C 接口_

##### USB-C 引脚定义

由于对称（除了 CC1、CC2）可以正负反着插

![USB-C 引脚](https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefinedTYPEC.png)
_USB-C 引脚定义_

| 引脚 | 说明 |
|------|------|
| D+/D- | 差分信号线，需要等长、等宽 |
| RX/TX | 串行总线信号线 |
| VBUS | 总线电压，为 USB-C 供电 |
| CC1/CC2 | PD 充电协议握手线 |
| TH | 外壳，接地即可 |

##### CC 引脚功能

- 确定是同向插进去还是反向插进去
- 确定供电和收电端
- 确定从机和主机能接受的电压

### 3.2 电源管理

#### LDO 工作原理

![LDO 原理](https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefinedLDO.png)
_LDO 工作原理_

若比基准电压高，MOS 关断一点，反之打开一点

由基准电压和电阻分压公式可计算 $V_{out}$

耗散功率 = $|V_{OUT}-V_{IN}| \cdot I_{LOAD}$

常用的 LDO：
- **AMS1117-3.3/AMS1117-5.0**：输出电压不可调
- **AMS1117-ADJ**：输出电压可调

#### DC-DC 工作原理

类似于 PWM，MOS 要么完全导通（MOS 上电压=0），要么完全截止（MOS 上电流=0），这两种状态 MOS 上都不会有功耗，效率高

常用型号：PS5430

#### 应用举例：给 STM32 供电

用 TPS5430 把 12V 降到 5V，再用 AMS1117 把 5V 降到 3.3V

**原因**：
- 直接用 AMS1117：功率过高，发热严重
- 直接用 TPS5430：会有波动导致工作不稳定

### 4.1 STM32 引脚

| BOOT1 | BOOT0 | 启动模式 | 说明 |

- **SWDIO**：双向数据线
- **SWCLK**：时钟线

## 五、通信协议基础

### 5.1 串口 (UART)

**UART** = Universal Asynchronous Receiver/Transmitter

**串行、异步、全双工**的通信协议

#### 特点

- **异步通信**：两个 uart 设备之间通信不需要时钟线，需要指定相同的传输速率
- **波特率**：每秒钟传送码元数量，波特率 = $\frac{比特率}{log_2N}$

#### 报文格式

![UART数据格式](https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefinedUART%E6%95%B0%E6%8D%AE%E6%A0%BC%E5%BC%8F.png)
_UART 报文格式_

串口报文由以下部分组成：

1. **起始位**（1bit）：逻辑"0"低电平，表示数据传输开始
2. **数据位**（5~8bit）：有效数据位数
3. **奇偶校验位**（1bit）：
   - 奇校验：有效数据和校验位中"1"的个数为奇数
   - 偶校验：有效数据和校验位中"1"的个数为偶数
   - 0 校验：校验位始终为 0
   - 1 校验：校验位始终为 1
4. **停止位**（1~1.5bit）：逻辑"1"高电平，表示传输结束
5. **空闲位**：发送完一组报文后，总线自动拉高到逻辑"1"

> [!NOTE]
> 接线的时候 TX 和 RX 要对着接

#### RS485 与 RS232

二者都是串口的衍生：

- **RS232**：电平转换芯片把 TTL 电平转化为 232 电平，增强抗干扰能力，传输距离更长
- **RS485**：把输入的串口信号转化为差分信号，A>B 表示 0，反之表示 1，抗干扰能力强，半双工

### 5.2 CAN 总线

**CAN** = Controller Area Network（控制器局域网），多用于汽车

单片机信号经过 CAN 收发芯片转化为差分信号

#### CAN 数据帧

![CAN 数据帧](https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefinedCAN.png)
_CAN 数据帧结构_

| 字段 | 说明 |
|------|------|
| 起始位 | 必须是逻辑 0 |
| 识别码 | 11 位，表明发给哪个设备，也代表优先级 |
| RTR 位 | 区分数据帧 (0) 和远程请求帧 (1) |
| 控制码 | 6 位，控制数据长度 |
| CRC 码 | 16 位，循环冗余校验位 |
| ACK 码 | 确认应答 |
| 结束位 | 7 位 1 |

### 5.3 I2C 总线

**I2C** = Inter-Integrated Circuit

一主多从，两根线：
- **SCL**：时钟线
- **SDA**：数据线

两根线都要加上拉电阻，空闲状态二者都是高电平

#### I2C 数据帧

![I2C 数据帧](https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefinediic.png)
_I2C 数据帧_

1. **起始位**：SCL 为高电平时，SDA 由高到低（下降沿）
2. **7 位设备地址码**
3. **读/写位**：写=0，读=1
4. **应答信号**：从机发送给主机。收到=0，未收到/读取接收完成=1
5. **八位寄存器地址**
6. **应答信号**
7. **八位数据**
8. **应答信号**
9. **停止位**：SCL 为高时，SDA 由低到高

### 5.4 SPI 总线

**SPI** = Serial Peripheral Interface（串行外设接口），高速、全双工

一主多从，四根线：

| 信号线 | 说明 |
|--------|------|
| SS | 片选，确定哪一个芯片通信，大多数低电平有效 |
| SCK | 时钟信号线，有空闲高/低，时钟上升/下降沿采集，共 4 种模式 |
| MOSI | 主设备输出，从设备输入 |
| MISO | 主设备输入，从设备输出 |

![SPI数据格式](https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefinedSPI%E6%95%B0%E6%8D%AE%E6%A0%BC%E5%BC%8F.png)

> [!TIP]
> **同步**：依赖于时钟
> **异步**：不依赖于时钟，而是依赖于二者约定好的频率
