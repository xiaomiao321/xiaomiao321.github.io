## STM32基础知识

### STM32系统框架

#### Cortex M内核 & 芯片

![0](https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefined0.png)

MCU厂商，经ARM公司授权，添加不同的外设 = 各种XX32芯片

#### STM32F1系统架构

4个主动单元 + 4个被动单元

| 主动单元(可以对被动单元发起通信控制) | 被动单元                          |
| ------------------------------------ | --------------------------------- |
| Cortex M3内核 DCode总线（D-Bus）     | 内部FLASH                         |
| Cortex M3内核 系统总线（S-Bus）      | 内部SRAM                          |
| 通用DMA1                             | FSMC                              |
| 通用DMA2                             | AHB到APB的桥，它连接的所有APB外设 |

其中，

- AHB=Advanced High Performance Bus：高级高性能总线，最高72MHz

- APB=Advanced Peripheral Bus：高级外围总线
    - APB1:最高36MHz
    - APB2：最高72MHz

- DMA=Direct Memory Access是数据搬运工，节省CPU资源

![1](https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefined1.png)

红色框是主动单元，蓝色框是被动单元

ICode总线直接连接Flash接口，不需要经过总线矩阵

#### STM32的寻址范围

32位的单片机可以有32根地址线，单片机内存地址访问的存储单元是按字节编址的

STM32寻址大小：2^32= 4G（字节）

STM32寻址范围：0x0000 0000 ~ 0xFFFF FFFF

#### 存储器映射

##### 基本概念

存储器指可以存储数据的设备，本身没有地址信息，对存储器分配地址的过程称为存储器映射

例如：

一个设备有19根地址线A0-A18，16根数据线D0-D15，则他的每个地址都是16位，即两个字节，一共有$2^{19}=512K$个地址，所以$总容量=512K · 2B = 1MB$，有不同的存储器映射方法：

映射1：0 - 512K

映射2：1 – 512K + 1

映射3：100K – 612K

映射4：512K – 1024K

##### STM32F1存储器的功能划分

###### 划分方式

ST将4GB（2^32）地址空间分成8个块

| **存储块**  | **功能**          | **地址范围**                       |
| ----------- | ----------------- | ---------------------------------- |
| **Block 0** | Code（FLASH）     | 0x0000 0000 ~ 0x1FFF FFFF（512MB） |
| **Block 1** | SRAM              | 0x2000 0000 ~ 0x3FFF FFFF（512MB） |
| **Block 2** | 片上外设          | 0x4000 0000 ~ 0x5FFF FFFF（512MB） |
| **Block 3** | FSMC Bank1&2      | 0x6000 0000 ~ 0x7FFF FFFF（512MB） |
| **Block 4** | FSMC Bank3&4      | 0x8000 0000 ~ 0x9FFF FFFF（512MB） |
| **Block 5** | FSMC寄存器        | 0xA000 0000 ~ 0xBFFF FFFF（512MB） |
| **Block 6** | 没用到            | 0xC000 0000 ~ 0xDFFF FFFF（512MB） |
| **Block 7** | Cortex M3内部外设 | 0xE000 0000 ~ 0xFFFF FFFF（512MB） |

###### Block0（FLASH）功能划分 

| **功能**                       | **地址范围**                       |
| ------------------------------ | ---------------------------------- |
| FLASH或系统存储器别名区        | 0X0000 0000 ~ 0x0007 FFFF（512KB） |
| 保留                           | 0X0008 0000 ~ 0x07FF FFFF          |
| 用户FLASH，用于存储用户代码    | 0X0800 0000 ~ 0x0807 FFFF（512KB） |
| 保留                           | 0X0808 0000 ~ 0x1FFF EFFF          |
| 系统存储器，存储出厂Bootloader | 0X1FFF F000 ~ 0X1FFF F7FF（2KB）   |
| 选项字节，配置读保护等         | 0X1FFF F800 ~ 0X1FFF F80F（16B）   |
| 保留                           | 0X1FFF F810 ~ 0X1FFF FFFF          |

###### Block1（SRAM）功能划分

| **功能** | **地址范围**                      |
| -------- | --------------------------------- |
| SRAM     | 0X2000 0000 ~ 0x2000 FFFF（64KB） |
| 保留     | 0X2001 0000 ~ 0x3FFF FFFF         |

###### Block2（外设）功能划分 （重要）

可以通过向对应地址写数据来操作外设，详见[寄存器映射](####寄存器映射)

| **功能**     | **地址范围**              |
| ------------ | ------------------------- |
| APB1总线外设 | 0X4000 0000 ~ 0x4000 77FF |
| 保留         | 0X4000 7800 ~ 0x4000 FFFF |
| APB2总线外设 | 0X4001 0000 ~ 0x4000 3FFF |
| 保留         | 0X4001 4000 ~ 0x4001 7FFF |
| AHB总线外设  | 0X4001 8000 ~ 0x4002 33FF |
| 保留         | 0X4002 3400 ~ 0x5FFF FFFF |

##### STM32F1的寄存器映射

寄存器是特殊的存储器，给寄存器地址命名的过程，就叫寄存器映射

![2](https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefined2.png)

例如，

```c
#define    GPIOA_ODR  	*(unsigned int *)(0x4001 080C) // 使用宏定义进行寄存器映射
GPIOA_ODR = 0XFFFF // 向对应寄存器写入16个1
```

###### 寄存器地址计算

为了方便编写代码及使用，我们将寄存器地址分为三个部分：

- 总线基地址（BUS_BASE_ADDR）
- 外设基于总线基地址的偏移量（PERIPH_OFFSET）
- 寄存器相对外设基地址的偏移量（REG_OFFSET）

寄存器地址 = BUS_BASE_ADDR +  PERIPH_OFFSET + REG_OFFSET

###### 总线基地址

| **总线** | **基地址**  | **偏移量** |
| -------- | ----------- | ---------- |
| **APB1** | 0X4000 0000 | 0          |
| **APB2** | 0X4001 0000 | 0X1 0000   |
| **AHB**  | 0X4001 8000 | 0X1 8000   |

APB1总线的基地址，也叫外设基地址（PERIPH_BASE）

此表的偏移量是相对外设基地址（PERIPH_BASE）来说的

###### GPIO外设基地址

| 所属总线          | **外设** | **基地址**  | **偏移量** |
| ----------------- | -------- | ----------- | ---------- |
| APB2(0X4001 0000) | GPIOA    | 0X4001 0800 | 0X800      |
| 同上              | GPIOB    | 0X4001 0C00 | 0XC00      |
| 同上              | GPIOC    | 0X4001 1000 | 0X1000     |
| 同上              | GPIOD    | 0X4001 1400 | 0X1400     |
| 同上              | GPIOE    | 0X4001 1800 | 0X1800     |
| 同上              | GPIOF    | 0X4001 1C00 | 0X1C00     |
| 同上              | GPIOG    | 0X4001 2000 | 0X2000     |

此表的偏移量是相对APB2外设基地址（APB2PERIPH_BASE）来说的

## HAL库

### 初识HAL库

#### CMSIS

CMSIS (微控制器软件接口标准)：Cortex Microcontroller Software Interface Standard，是由ARM和与其合作的芯片厂商、软件工具厂商，共同制定的标准

![3](https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefined3.png)

作用：增强ARM产品的兼容性

#### HAL库

ST 为了方便用户开发 STM32芯片开发提供了三种库：

- 标准外设库 (Standard Peripheral Libraries)
- HAL库(硬件抽象层)：Hardware Abstraction Layer
- LL库(底层库）：Low Layer

开发方法

- 直接操作寄存器：执行效率高时间成本高

- 标准库：最早的，但F0/F1/F3/F2/F4/L1目前已停止维护

- HAL库：全系列兼容，ST目前主推的库。兼容性好、易移植性好、效率低

- LL库：全系列兼容与HAL库捆绑发布轻量级、效率高不匹配部分复杂外设，如USB

HAL库回调函数(主要为了管理代码)

回调函数：回调函数是一种特殊的函数，它作为参数传递给另一个函数，并在被调用函数执行完毕后被调用

例如，假设有一个回调函数需要接收两个整数参数并返回一个整数值，可以使用以下方式定义函数指针：

```c
int (*callback)(int, int);
```

然后，可以将一个实际的函数指针赋值给它，例如：

```c
int add(int a, int b) {
    return a + b;
}
callback = add;
```

现在，可以将这个函数指针传递给其他函数，使得其他函数可以使用这个指针来调用该函数。

| **回调函数**                     | **举例**                                                                                                                                                     |
| -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| HAL_PPP_MspInit() / _MspDeInit() | 举例: HAL_USART_MspInit()会被HAL_PPP_Init() 函数调用，该函数主要用于存放外设使用到的GPIO、CLOCK、NVIC、DMA等初始化代码                                       |
| **HAL_PPP_ProcessCpltCallback**  | 举例: HAL_USART_TxCpltCallback由外设中断或DMA中断调用，HAL库中断公共处理函数已经实现对中断标记位读取、判断和清除操作，用户只需要专注于中断逻辑功能的实现即可 |
| **HAL_PPP_ErrorCallback**        | 举例: HAL_USART_ErrorCallback外设或DMA中断中发生的错误，用于发生错误处理                                                                                     |

此类函数通常被_weak修饰（弱函数），允许用户重新定义该函数

MAP文件是MDK编译代码后，产生的集程序、数据及IO空间的一种映射列表文件，用于分析各.c文件占用FLASH 和 RAM的大小，方便优化代码

## STM32启动

### STM32启动模式（也称自举模式）

M3/M4/M7等内核复位后，做的第一件事：

1，从地址 0x0000 0000 处取出堆栈指针 MSP 的初始值，该值就是栈顶地址

2，从地址 0x0000 0004 处取出程序计数器指针 PC 的初始值，该值是复位向量

因为是32位单片机，所以每次是4个字节，地址值加四

为了确定启动模式，因为芯片厂商可以把0x0000 0000和0x0000 0004地址映射到其它的地址。

在系统复位后，SYSCLK的第4个上升沿，BOOT引脚的值将被锁存

STM32启动过程（内部FLASH启动为例）

#### 启动文件介绍

1，初始化MSP 从0X0800 0000获取

2，初始化PC 从0X0800 0004获取

3，设置堆栈大小 Heap_Size(堆)、Stack_Size(栈)

4，初始化中断向量表 __Vectors定义

5，调用初始化函数 可选的，如调用： SystemInit函数

6，调用__main 标准C库函数，执行一系列设置，最终调用main函数

#### Reset_Handler函数介绍

```assembly
Reset_Handler   PROC
                EXPORT  Reset_Handler   [WEAK]
                IMPORT  __main
                IMPORT  SystemInit
                LDR   R0, =SystemInit
                BLX   R0               
                LDR   R0, =__main
                BX      R0
                ENDP
```

EXPORT：标明全局属性，可被外部调用IMPORT：申明来自外部文件，类似extern

PROC：定义子程序

ENDP：表示子程序结束

WEAK：弱定义，允许用户重定义

> [!NOTE]
>
> 如果外部没有SystemInit函数则会报错！

SystemInit函数会在HAL库定义

##  时钟树

### 时钟树简介

#### 时钟简介

![15](https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefined15.png)

**时钟是单片机运行的基础，时钟信号推动单片机内各个部分执行相应的指令。**时钟系统就是CPU的脉搏，决定cpu速率，像人的心跳一样 只有有了心跳，人才能做其他的事情，而单片机有了时钟，才能够运行执行指令，才能够做其他的处理 (点灯，串口，ADC），时钟的重要性不言而喻。简单来说，时钟是具有周期性的脉冲信号，最常用的是占空比50%的方波

STM32本身十分复杂，外设非常多  但我们实际使用的时候只会用到有限的几个外设，使用任何外设都需要时钟才能启动，但并不是所有的外设都需要系统时钟那么高的频率，为了兼容不同速度的设备，有些高速，有些低速，如果都用高速时钟，势必造成浪费   并且，同一个电路，时钟越快功耗越快，同时抗电磁干扰能力也就越弱，所以较为复杂的MCU都是采用多时钟源的方法来解决这些问题。所以便有了STM32的时钟系统和时钟树

![16](https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefined16.png)

![17](https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefined17.png)

A=1,B=1，输出0

A=1,B=0，输出0

B从1到0转化的过程中，异或门下边的输入会瞬间变0，而上面的输入由于与门的存在会延迟一小会变成0，中间有一段时间是1和0异或输出1，会产生脉冲(毛刺、尖峰)，不好。

加入时钟，低电平时切换电平，在高电平采集，从而确保信号是确定的值

void sys_stm32_clock_init(uint32_t plln)
{
    HAL_StatusTypeDef ret = HAL_ERROR;
    RCC_OscInitTypeDef rcc_osc_init = {0};// 初始化，不配置的默认设成0
    RCC_ClkInitTypeDef rcc_clk_init = {0};

    rcc_osc_init.OscillatorType = RCC_OSCILLATORTYPE_HSE;       /* 选择要配置HSE */
    rcc_osc_init.HSEState = RCC_HSE_ON;                         /* 打开HSE */
    rcc_osc_init.HSEPredivValue = RCC_HSE_PREDIV_DIV1;          /* HSE预分频系数 */
    rcc_osc_init.PLL.PLLState = RCC_PLL_ON;                     /* 打开PLL(锁相环) */
    rcc_osc_init.PLL.PLLSource = RCC_PLLSOURCE_HSE;             /* PLL时钟源选择HSE */
    rcc_osc_init.PLL.PLLMUL = plln;                             /* PLL倍频系数 */
    ret = HAL_RCC_OscConfig(&rcc_osc_init);                     /* 初始化 */

    if (ret != HAL_OK)
    {
        while (1);                                              /* 时钟初始化失败，之后的程序将可能无法正常执行，可以在这里加入自己的处理 */
    }
    /*====================================================================*/
        /* 选中PLL作为系统时钟源并且配置HCLK,PCLK1和PCLK2*/
    rcc_clk_init.ClockType = (RCC_CLOCKTYPE_SYSCLK | RCC_CLOCKTYPE_HCLK | RCC_CLOCKTYPE_PCLK1 | RCC_CLOCKTYPE_PCLK2);
    rcc_clk_init.SYSCLKSource = RCC_SYSCLKSOURCE_PLLCLK;        /* 设置系统时钟来自PLL */
    rcc_clk_init.AHBCLKDivider = RCC_SYSCLK_DIV1;               /* AHB分频系数为1 */
    rcc_clk_init.APB1CLKDivider = RCC_HCLK_DIV2;                /* APB1分频系数为2 */
    rcc_clk_init.APB2CLKDivider = RCC_HCLK_DIV1;                /* APB2分频系数为1 */
    ret = HAL_RCC_ClockConfig(&rcc_clk_init, FLASH_LATENCY_2);  /* 同时设置FLASH延时周期为2WS，也就是3个CPU周期。 */

    if (ret != HAL_OK)
    {
        while (1);                                              /* 时钟初始化失败，之后的程序将可能无法正常执行，可以在这里加入自己的处理 */
    }

#### SysTick工作原理

SysTick，即系统滴答定时器，包含在M3/4/7内核里面，核心是一个24位的递减计数器

每来一个时钟减一

> [!NOTE]
>
> 每次VAL到0时，VAL自动从LOAD重载，开始新一轮递减计数，LOAD可自行设置0-$2^{24}-1$

#### SysTick寄存器介绍

##### SysTick控制及状态寄存器(CTRL)

| 位段 | 名称 | 类型 | 复位值 | 描述 |
|:---:|:---|:---:|:---:|:---|
| 16 | COUNTFLAG | R | 0 | 如果在上次读取本寄存器后，SysTick 已经数到了0，则该位为 1。如果读取该位，该位将自动清零 |
| 2 | CLKSOURCE | R/W | 0 | 0=外部时钟源(STCLK)<br>1=内核时钟(FCLK) |
| 1 | TICKINT | R/W | 0 | 1=SysTick 倒数到 0 时产生 SysTick 异常请求<br>0=数到 0 时无动作 |
| 0 | ENABLE | R/W | 0 | SysTick 定时器的使能位 |

`CLKSOURCE`已被重新设为0=8分频，1=1分频

`TICKINT`=1数到零进中断

##### SysTick重装载数值寄存器(LOAD)

| 位段 | 名称 | 类型 | 复位值 | 描述 |
|:---:|:---|:---:|:---:|:---|
| **23:0** | RELOAD | R/W | 0 | 当倒数至零时，将被重装载的值 |

##### SysTick当前数值寄存器(VAL)

| 位段 | 名称 | 类型 | 复位值 | 描述 |
|:---:|:---|:---:|:---:|:---|
| **23:0** | CURRENT | R/Wc | 0 | 读取时返回当前倒计数的值，写它则使之清零，同时还会清除在 SysTick 控制及状态寄存器中的COUNTFLAG 标志 |

#### delay_init函数

```c
void delay_init(uint16_t sysclk) // sysclk的单位是MHz
{ 
	SysTick->CTRL = 0; // 把HAL库自动配置的清空
    // 选择8分频，72/8=9M是实际计数频率
	HAL_SYSTICK_CLKSourceConfig(SYSTICK_CLKSOURCE_HCLK_DIV8); 
    // 得到1us要数的次数 = sysclk / 8 = 9，即g_fac_us
	g_fac_us = sysclk / 8; // 全局变量：1us时基的来源 
}
```

#### delay_us()函数

```c
void delay_us(uint32_t nus) 
{ 
	uint32_t temp; 
	SysTick->LOAD = nus * g_fac_us; 	/* 时间加载 多少个9次就是多少个1us*/ 
	SysTick->VAL = 0x00; 			/* 清空计数器 */ 
	SysTick->CTRL |= 1 << 0 ; 		/* 开始倒数 */ 
	do 
	{ 
		temp = SysTick->CTRL; 
	} while ((temp & 0x01) && !(temp & (1 << 16))); /* CTRL.ENABLE位必须为1, 并等待时间到达 寄存器的第16位表明了是否计时完毕，上面提及过*/

 
	SysTick->CTRL &= ~(1 << 0) ; 		/* 关闭SYSTICK */ 
	SysTick->VAL = 0X00; 			/* 清空计数器 */ 
}

```

$$
最大时间 = \frac{2^{24}}{9·10^6}=1.864s
$$



#### delay_ms()函数

```c
void delay_ms(uint16_t nms) 
{ 
	uint32_t repeat = nms / 1000;	/* 这里用1000,是考虑到可能有超频应用, 
							    	 * 比如128Mhz的时候, delay_us最大只能延时1000ms
								 */ 
	uint32_t remain = nms % 1000; // 尾数
	while (repeat) // 大于1000ms
	{ 
		delay_us(1000 * 1000); 	/* 利用delay_us 实现 1000ms 延时 */ 
		repeat--; 
	} 
	if (remain) 
	{ 
		delay_us(remain * 1000); 	/* 利用delay_us, 把尾数延时(remain ms)给做了 */ 
	} 
}
```

##### ODR和BSRR寄存器控制输出有什么区别？

使用ODR，在读和修改访问之间产生中断时，可能会发生同时操作同一个寄存器的风险；BSRR则无风险。

```c
GPIOB->ODR |= 1 << 3; //ODR = ODR | (1<<3) 需要先读                 /* PB3 = 1 */

GPIOB->BSRR = 0x00000008;      /* PB3 = 1 */
```

ODR修改：读->改->写
BSRR修改：直接写

## 中断

### 什么是中断

#### 中断的定义

打断CPU执行正常的程序，转而处理紧急程序，然后返回原暂停的程序继续运行，就叫中断。

比如做作业的时候门铃响了去开门，开完门继续做作业

#### 中断的作用和意义

##### 作用

###### 实时控制

在确定时间内对相应事件作出响应，如：温度监控

###### 故障处理

检测到故障，需要第一时间处理，如：电梯门夹人了

###### 数据传输

不确定数据何时会来，如：串口数据接收

##### 意义

高效处理紧急程序，不会一直占用CPU资源

### NVIC

#### NVIC的基本概念

Nested vectored interrupt controller，嵌套向量中断控制器，属于内核（M3/4/7）

NVIC支持：256个中断（16内核 + 240外部），支持：256个优先级，允许芯片厂商裁剪：

| STM32型号   | 内核中断 | 外部中断 | 中断优先级 |
| ----------- | -------- | -------- | ---------- |
| STM32F103xx | 10       | 60       | 16         |
| STM32F407xx | 10       | 82       | 16         |
| STM32F429xx | 10       | 91       | 16         |
| STM32F750xx | 10       | 98       | 16         |
| STM32F767xx | 10       | 110      | 16         |
| STM32H743xx | 10       | 150      | 16         |
| STM32H750xx | 10       | 150      | 16         |

#### 中断向量表

定义一块固定的内存，以4字节对齐，存放各个中断服务函数程序的首地址，有了这个地址就能找到中断服务函数，相当于函数指针

#### STM32中断优先级基本概念

- 抢占优先级(pre)：高抢占优先级可以打断正在执行的低抢占优先级中断
- 响应优先级(sub)：当抢占优先级相同时，响应优先级高的先执行，但是**不能互相打断**
- 抢占和响应都相同的情况下，自然优先级越高的，先执行
- 自然优先级：中断向量表的优先级

> [!NOTE]
>
> 数值越小，表示优先级越高

#### STM32中断优先级分组

| **优先级分组** | **AIRCR[10:8]** | **IPR****x** **bit[7:4]****分配** | **分配结果**                 |
| -------------- | --------------- | --------------------------------- | ---------------------------- |
| **0**          | 111             | None ：[7:4]                      | 0位抢占优先级，4位响应优先级 |
| **1**          | 110             | [7] ：[6:4]                       | 1位抢占优先级，3位响应优先级 |
| **2**          | 101             | [7:6] ：[5:4]                     | 2位抢占优先级，2位响应优先级 |
| **3**          | 100             | [7:5] ：[4]                       | 3位抢占优先级，1位响应优先级 |
| **4**          | 011             | [7:4] ：None                      | 4位抢占优先级，0位响应优先级 |

> [!NOTE]
>
> 一个工程中，一般只设置一次中断优先级分组。

以分组是2为例

| **编号** | **自然优先级** | **对应外设** | **抢占** | **响应** | **执行顺序** |
| -------- | -------------- | ------------ | -------- | -------- | ------------ |
| **3**    | 10             | RTC          | 2        | 1        | 2            |
| **6**    | 13             | EXTI0        | 3        | 0        | 4            |
| **7**    | 14             | EXTI1        | 2        | 0        | 1            |
| **-1**   | 6              | Systick      | 3        | 0        | 3            |

则EXTI1和RTC可以打断EXTI0和Systick的中断，获得优先执行

### EXTI

#### EXTI的基本概念

External(Extended) interrupt/event Controller，外部(扩展)中断事件控制器，管理着内外部的中断和事件包含20个产生事件/中断请求的边沿检测器，即总共：20条EXTI线（F1）

![54](https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefined54.png)

#### 中断和事件的理解

中断：进入NVIC，有相应的中断服务函数，需要CPU处理

事件：不进入NVIC，仅用于内部硬件自动控制的，如：TIM进行一次波形输出、DMA搬运一次数据、ADC进行一次数模转换

#### EXTI支持的外部中断/事件请求

| **中断线**                     | **F1** | **F4** | **F7** |
| ------------------------------ | ------ | ------ | ------ |
| EXTI线0~15：对应GPIO PIN 0~15  | ✔      | ✔      | ✔      |  |
| EXTI线16：PVD输出              | ✔      | ✔      | ✔      |  |
| EXTI线17：RTC闹钟事件          | ✔      | ✔      | ✔      |  |
| EXTI线18：USB OTG FS唤醒事件   | ✔      | ✔      | ✔      |  |
| EXTI线19：以太网唤醒事件       |        | ✔      | ✔      |  |
| EXTI线20：USB OTG HS唤醒事件   |        | ✔      | ✔      |  |
| EXTI线21：RTC 入侵和时间戳事件 |        | ✔      | ✔      |  |
| EXTI线22：RTC 唤醒事件         |        | ✔      | ✔      |  |
| EXTI线23：LPTIM1 异步事件      |        |        | ✔      |  |
| …                              |        |        |        |  |

0-15比较重要

#### EXTI的主要特性

每条EXTI线都可以单独配置：选择类型（中断或者事件）、触发方式（上升沿，下降沿或者双边沿触发）、支持软件触发、开启/屏蔽、有挂起状态位

baud=\frac{f_{ck}}{16∗USARTDIV}
$$


其中$f_{ck}$是串口的时钟，如USART1的时钟是APB2总线时钟，最高稳定频率72MHz，其他串口都是APB1总线时钟

$f_{ck}$和波特率都已知，用上面公式可计算USARTDIV，从何得到DIV_Mantissa和DIV_Fraction

DIV_Mantissa是USARTDIV的整数部分，DIV_Fraction是USART的小数部分，写入BRR寄存器

##### 波特比率寄存器（BRR）

![75](https://photos-1355819942.cos.ap-shanghai.myqcloud.com/undefined75.png)

把USARTDIV的整数部分写入位[15:4]，小数部分写入位[3:0]，就可以配置波特率

以波特率115200，USART1为例，USARTDIV=39.0625

写入BRR寄存器:

```c
uint16_t mantissa;
uint16_t fraction;
mantissa = 39;
fraction = 0.0625 * 16 + 0.5=0x01
    /*+0.5是为了四舍五入，因为uint6_t会把小数部分直接砍掉*/
USART1->BRR = (mantissa<<4)+fraction;
/*左移四位是因为整数部分是从第四位开始的*/
```

简化版：

```c
USART1->BRR = USARTDIV * 16 + 0.5; /*相当于把前面左移4位(乘16)和小数部分*16+0.5合并了*/
```

最终版:

$$
USART->BRR \\
=USARTDIV * 16 + 0.5 \\
=\frac{f_{ck}}{16*baud}*16+0.5 \\
=\frac{f_{ck}}{baud}+0.5
$$


用HAL库直接传参数即可

```c
void HAL_UART_IRQHandler(UART_HandleTypeDef *huart)
{
    /* 如果没有发生错误 */
    errorflags = (isrflags & (uint32_t)(USART_SR_PE | USART_SR_FE | USART_SR_ORE | USART_SR_NE));
    if (errorflags == RESET)
    {
        /* UART接收模式。 RXNE:读数据寄存器非空  RXNEIE:接收缓冲区非空中断使能 */
        if (((isrflags & USART_SR_RXNE) != RESET) && ((cr1its & USART_CR1_RXNEIE) != RESET))
        {
            UART_Receive_IT(huart);  /* 在该函数里清楚相关中断标志位并调用HAL_UART_RxCpltCallback */
            return;
        }
    }

    /* 如果发生了错误 */
    if ((errorflags != RESET) ...)  {...}

    /* UART发送模式。 TXE:发送数据寄存器空  TXEIE: 发送缓冲区空中断使能 */
    if (((isrflags & USART_SR_TXE) != RESET) && ((cr1its & USART_CR1_TXEIE) != RESET))
    {
        UART_Transmit_IT(huart);
        return;
    }
    /* UART发送模式结束。 TC: 发送完成   TCIE: 发送完成中断使能 */
    if (((isrflags & USART_SR_TC) != RESET) && ((cr1its & USART_CR1_TCIE) != RESET))
    {
        UART_EndTransmit_IT(huart);  /* 在该函数里清楚相关中断标志位并调用HAL_UART_TxCpltCallback */
        return;
    }
}

##### 中断模式

配置步骤：NVIC-USART2 global interrupt

轮询模式原理：发送移位寄存器发送完成后，发送数据寄存器中的数据会被移入发送移位寄存器进行发送。在此过程中，CPU需要不断查询发送数据寄存器中的数据是否已经移动，移动了就把下个数据放进来，否则不停查询，直到都发完或者超时

接收移位寄存器->接收数据寄存器，CPU一直查是否有数据可以读，导致堵塞。使用串口中断模式CPU可以把数据放入寄存器之后进行其他任务，发送移位寄存器发完之后会触发中断，CPU在中断处理函数把数据放入发送数据寄存器后可以处理其它代码。

既然不堵塞了，怎么知道接收完成？接收移位寄存器把一帧数据移入接收数据寄存器会触发接收数据寄存器非空中断，但是每个UART只有一个中断向量，中断处理函数不一定是接收数据寄存器非空中断调用的，还可能是发送数据寄存器空中断等，需要进行判断，判断已被HAL库实现，我们编写回调函数即可

## IWDG

### IWDG简介

- 全称：Independent watchdog，即独立看门狗

- 本质：能产生系统复位信号的计数器

- 特性：
    - 递减的计数器时钟由独立的RC振荡器提供（可在待机和停止模式下运行，不来自主时钟）

    - 看门狗被激活后，当递减计数器计数到0x000时产生复位

- 喂狗：在计数器计数到0之前，重装载计数器的值，防止复位

#### STM32系统复位

- NRST引脚上的低电平
- 窗口看门狗计数终止[WWDG复位](##WWDG)
- 独立看门狗计数终止(IWDG复位)
- 软件复位(SW复位)
- 低功耗管理复位

### IWDG作用

#### 单片机的异常情况

外界电磁干扰或者自身系统（硬件或软件）异常，造成程序跑飞，如：陷入某个不正常的死循环，打断正常的程序运行。通常表现为死机

#### IWDG作用

主要用于检测外界电磁干扰，或硬件异常导致的程序跑飞问题(软件由WWDG检测)

#### IWDG应用

在一些需要高稳定性的产品中，并且对时间精度要求较低的场合

> [!NOTE]
>
> 独立看门狗是异常处理的最后手段，不可依赖，应在设计时尽量避免异常的发生

#### 寄存器配置操作步骤

- 通过在键寄存器 (IWDG_KR) 中写入 0xCCCC 来使能 IWDG。
- 通过在键寄存器 (IWDG_KR) 中写入 0x5555 来使能寄存器访问。

- 刷新计数器值为 IWDG_RLR 的值 (IWDG_KR = 0xAAAA)。

## WWDG

### WWDG简介

- 全称：Window watchdog，即窗口看门狗

- 本质：能产生**系统复位信号**和**提前唤醒中断**的计数器

- 特性：

    - 递减的计数器

    - 当递减计数器值从 0x40减到0x3F时复位（即T6位跳变到0）

    - 计数器的值大于W[6:0]值时喂狗会复位

    - 提前唤醒中断 (EWI)：当递减计数器等于 0x40 时可产生

- 喂狗：在窗口期内重装载计数器的值，防止复位

### IWDG作用

#### IWDG作用

用于监测单片机程序运行时效是否精准，主要检测软件异常

#### IWDG应用

需要精准检测程序运行时间的场合，因为时钟源来自系统时钟

### WWDG工作原理

![image-20250709223456827](C:\Users\xiaom\AppData\Roaming\Typora\typora-user-images\image-20250709223456827.png)

- 计数器计数值最大值127，可由用户设置
- 到达窗口上限值(可由用户设置)，在窗口上限值之前喂狗会复位
- 减到0x40可以产生中断
- 到窗口下限值会复位
- W[6:0] ≥窗口期＞0x3F

> [!NOTE]
>
> W[6:0] 必须大于窗口下限值0x3F，否则无窗口期

### 通用定时器

#### 通用定时器简介

通用定时器：TIM2/TIM3 /TIM4 /TIM5

主要特性：

- 16位递增、递减、中心对齐计数器（计数值：0-65535）
- 16位预分频器（分频系数：1~65536）
- 可用于触发DAC、ADC
- 在更新事件、触发事件、输入捕获、输出比较时，会产生中断/DMA请求。基本定时器只具备第一个
- 4个独立通道，可用于：输入捕获、输出比较、输出PWM、单脉冲模式。基本定时器无独立通道
- 使用外部信号控制定时器且可实现多个定时器互连的同步电路，即定时器的级联：用一个定时器的溢出事件作为下一个定时器的时钟源
- 支持编码器和霍尔传感器电路等，多用于电机

 *       注意:为了通用,我们默认ARR和CCRy都是16位寄存器,对于32位的定时器(如:TIM5),也只按16位使用

/* 定时器输入捕获中断处理回调函数 */
void HAL_TIM_IC_CaptureCallback(TIM_HandleTypeDef *htim)
{
    if (htim->Instance == TIM5)
    {
        if ((g_timxchy_cap_sta & 0X80) == 0)                /* 还没有成功捕获 */
        {
            if (g_timxchy_cap_sta & 0X40)                   /* 捕获到一个下降沿 */
            {
                g_timxchy_cap_sta |= 0X80;                  /* 标记成功捕获到一次高电平脉宽 */
                g_timxchy_cap_val = HAL_TIM_ReadCapturedValue(&g_timx_cap_chy_handle, TIM_CHANNEL_1);  /* 获取当前的捕获值 */
                TIM_RESET_CAPTUREPOLARITY(&g_timx_cap_chy_handle, TIM_CHANNEL_1);                      /* 一定要先清除原来的设置 */
                TIM_SET_CAPTUREPOLARITY(&g_timx_cap_chy_handle, TIM_CHANNEL_1, TIM_ICPOLARITY_RISING); /* 配置TIM5通道1上升沿捕获 */
            }
            else /* 还未开始,第一次捕获上升沿 ,计数器清零，同时配置为下降沿捕获*/
            {
                g_timxchy_cap_sta = 0;                              /* 清空 */
                g_timxchy_cap_val = 0;
                g_timxchy_cap_sta |= 0X40;                          /* 标记捕获到了上升沿 */
                __HAL_TIM_DISABLE(&g_timx_cap_chy_handle);          /* 关闭定时器5 */
                __HAL_TIM_SET_COUNTER(&g_timx_cap_chy_handle, 0);   /* 定时器5计数器清零 */
                TIM_RESET_CAPTUREPOLARITY(&g_timx_cap_chy_handle, TIM_CHANNEL_1);   /* 一定要先清除原来的设置！！ */
                TIM_SET_CAPTUREPOLARITY(&g_timx_cap_chy_handle, TIM_CHANNEL_1, TIM_ICPOLARITY_FALLING); /* 定时器5通道1设置为下降沿捕获 */
                __HAL_TIM_ENABLE(&g_timx_cap_chy_handle);           /* 使能定时器5 */
            }
        }
    }
}

/* 定时器更新中断回调函数 */
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim)
{
    if (htim->Instance == TIM5)
    {
        if ((g_timxchy_cap_sta & 0X80) == 0)            /* 还未成功捕获 */
        {
            if (g_timxchy_cap_sta & 0X40)               /* 已经捕获到高电平了 */
            {
                if ((g_timxchy_cap_sta & 0X3F) == 0X3F) /* 高电平太长了 */
                {
                    TIM_RESET_CAPTUREPOLARITY(&g_timx_cap_chy_handle, TIM_CHANNEL_1);                     /* 一定要先清除原来的设置 */
                    TIM_SET_CAPTUREPOLARITY(&g_timx_cap_chy_handle, TIM_CHANNEL_1, TIM_ICPOLARITY_RISING);/* 配置TIM5通道1上升沿捕获 */
                    g_timxchy_cap_sta |= 0X80;          /* 标记成功捕获了一次 */
                    g_timxchy_cap_val = 0XFFFF;			/*超时了，设置为最大*/
                }
                else      /* 累计定时器溢出次数 */
                {
                    g_timxchy_cap_sta++;
                }
            }
        }
    }
}

##### 重复计数器特性

计数器每次上溢或下溢都能使重复计数器减1，减到0时，再发生一次溢出就会产生更新事件如果设置RCR为N，更新事件将在N+1次溢出时发生

![image-20250712173836251](C:\Users\xiaom\AppData\Roaming\Typora\typora-user-images\image-20250712173836251.png)

第N+1次溢出的时候发生更新事件

中心对齐：CNT=ARR-1或CNT=1

再同步：相当于重置影子寄存器

##### 高级定时器输出指定个数PWM原理

1，配置边沿对齐模式输出PWM

2，指定输出N个PWM，则把N-1写入RCR

3，在更新中断内，关闭计数器

#### 外部时钟

内部时钟的：内部时钟->触发控制器->预分频器->计数器<-自动重装载寄存器。

把内部时钟改成GPIO口即可实现外部时钟的计数。

每个定时器有4个外部通道(TI1-TI1,TI=TIM Input)，3和4没有接入到触发控制器->滤波器->边沿检测器(上升沿、下降沿、双边沿)-›TI1FP1、TI1FP2、TI2FP1、TI2FP2。TI1FP1和TI2FP2来到了触发控制器，通道一还有TI1_ED也到了触发控制器，它只能是双边沿，前两个上升、下降、双边沿都行，还有一个ETR也到了触发控制器触发器选择一个信号->触发控制器->从模式控制器(外部时钟模式一)

外部时钟模式二：ETR独立进入触发控制器，不经过触发器

配置输入模式：外部时钟模式二直接选来源是ETR即可，外部时钟模式一需要在从模式选外部时钟模式一

输入滤波器：N次检验后脉冲依然没结束，判定该脉冲有效，输出脉冲信号到定时器；否则忽视此脉冲，大多数直接设置成15即可

> 这个滤波器的采样频率与内部时钟频率有关，可以通过设置APB预分频器来保证输入到定时器的频率不要太快来让主频保持72MHz

#### 输入捕获(测量脉冲长度)

超声波模块原理：有Trig和Echo引脚，向Trig发送脉冲信号，超声波模块接收到后会向外发送超声波，然后超声波模块会将Echo拉高，接收到反射回来的超声波后会将Echo拉低，其上高电平的时间就是超声波反射的时间

当捕获到边沿时，立刻把计数器的值写入捕获寄存器，等待被读取

#### 旋转编码器

增量型旋转编码器：有A，B两相输出信号，旋转编码器未检测到旋转时，二者均无电平变化；

顺时针旋转时，二者都输出方波，B相领先A相0.25个周期(A上升沿，B高电平)

顺时针旋转时，二者都输出方波，B相落后A相0.25个周期(A上升沿，B低电平)

![e43e6759df9fd76462f06907132765ee](D:\xwechat_files\wxid_85qaa11voxkj32_0996\temp\RWTemp\2025-07\9e20f478899dc29eb19741386f9343c8\e43e6759df9fd76462f06907132765ee.jpg)

简单思路：A上升沿触发中断，中断回调函数读取B的电平，根据B的电平加或减计数值。但是对于太快的不行，采用定时器A作为外部输入。定时器有专门的编码器接口，把A、B信号输入进去即可实现正传自增，翻转自减。

> [!NOTE]
>
> 编码器接口对上下边沿都敏感，对于A、B上的一组脉冲，会计数两次，需要二分频

编码器输入接口是TI1FP1和TI2FP2，将A、B信号接入定时器的通道1和通道2,即可接入到编码器接口，根据A、B信号对计数器进行加减操作。同时还可以用所在通道进入时的输入滤波器和边沿检测器进行初步处理。

配置：无须选时钟源，combined channels-encoder mode即可

调整极性可以改变顺时针或逆时针与加或减的对应关系，第一个调成Falling edge是顺时针增加，逆时针减少

#### 直流电机

一端通正极，一端通负极即可旋转。用电机驱动芯片，因为GPIO驱动能力不足，以及电机的反电动势

DRV8833电机驱动芯片可以同时驱动两个电机

单片机输出PWM到IN1,IN2控制OUT1,OUT2的输出，来控制电机旋转，IN3,IN4同理

IN1高电平，IN2低电平：OUT1高电平，OUT2低电平，正转；反之亦然

IN1高电平，IN2高电平：OUT1=OUT2=低电平：(电流)慢衰减，也称为刹车

IN1=IN2低电平：OUT1=OUT2=断路：快衰减，也称为滑行

IN1替换为PWM，则电机一会正转，一会滑行，占空比越高，速度越快，称为快衰减

IN2替换为PWM，则电机一会正转，一会刹车，占空比越低，速度越快，称为慢衰减![a6b3d63bb6aa1c7d01ef4950adfcf4bc](D:\xwechat_files\wxid_85qaa11voxkj32_0996\temp\RWTemp\2025-07\9e20f478899dc29eb19741386f9343c8\a6b3d63bb6aa1c7d01ef4950adfcf4bc.jpg)

/**
 * @file drv8833.c
 * @brief DRV8833双H桥电机驱动器控制实现
 */

#include "drv8833.h"
#include "tim.h"

/** @brief IN1引脚的定时器 */
#define IN1_TIM &htim2
/** @brief IN2引脚的定时器 */
#define IN2_TIM &htim2

/** @brief IN1引脚的定时器通道 */
#define IN1_CH TIM_CHANNEL_1
/** @brief IN2引脚的定时器通道 */
#define IN2_CH TIM_CHANNEL_2

/** @brief 最大速度值 */
#define MAX_SPEED 100


/** @brief 当前衰减模式 */
static DecayMode currentDecayMode = SLOW_DECAY;

/**
 * @brief 设置IN1引脚的PWM占空比
 * @param duty 占空比值
 */
static inline void __SetIn1PWM(uint8_t duty)
{
    __HAL_TIM_SET_COMPARE(IN1_TIM, IN1_CH, duty);
}

/**
 * @brief 设置IN2引脚的PWM占空比
 * @param duty 占空比值
 */
static inline void __SetIn2PWM(uint8_t duty)
{
    __HAL_TIM_SET_COMPARE(IN2_TIM, IN2_CH, duty);
}

/**
 * @brief 初始化DRV8833
 */
void DRV8833_Init(void)
{
    HAL_TIM_PWM_Start(IN1_TIM, IN1_CH);
    HAL_TIM_PWM_Start(IN2_TIM, IN2_CH);
}

/**
 * @brief 设置衰减模式
 * @param mode 衰减模式
 */
void DRV8833_SetDecayMode(DecayMode mode)
{
    currentDecayMode = mode;
}

/**
 * @brief 控制电机前进
 * @param speed 速度值（0-100）
 */
void DRV8833_Forward(uint8_t speed)
{
    if (speed > MAX_SPEED)
        speed = MAX_SPEED;
    
    if (currentDecayMode == FAST_DECAY) {
        __SetIn1PWM(speed);
        __SetIn2PWM(0);
    } else {
        __SetIn1PWM(MAX_SPEED);
        __SetIn2PWM(MAX_SPEED - speed);
    }
}

/**
 * @brief 控制电机后退
 * @param speed 速度值（0-100）
 */
void DRV8833_Backward(uint8_t speed)
{
    if (speed > MAX_SPEED)
        speed = MAX_SPEED;
    
    if (currentDecayMode == FAST_DECAY) {
        __SetIn1PWM(0);
        __SetIn2PWM(speed);
    } else {
        __SetIn1PWM(MAX_SPEED - speed);
        __SetIn2PWM(MAX_SPEED);
    }
}

/**
 * @brief 电机刹车
 */
void DRV8833_Brake(void)
{
    __SetIn1PWM(MAX_SPEED);
    __SetIn2PWM(MAX_SPEED);
}

/**
 * @brief 电机滑行
 */
void DRV8833_Coast(void)
{
    __SetIn1PWM(0);
    __SetIn2PWM(0);
}

### 堆和栈

#### 栈

栈由操作系统自动分配释放 ，用于存放函数的参数值、局部变量

```c
int main() {
	int b;				//栈
	char s[] = "abc"; 	//栈
	char *p2;			//栈
}
```

其中函数中定义的局部变量按照先后定义的顺序依次压入栈中，也就是说相邻变量的地址之间不会存在其它变量。栈的内存地址生长方向与堆相反，由高到底，所以后定义的变量地址低于先定义的变量，比如上面代码中变量 s 的地址小于变量 b 的地址，p2 地址小于 s 的地址。栈中存储的数据的生命周期随着函数的执行完成而结束

#### 堆

堆由开发人员分配和释放， 若开发人员不释放，程序结束时由 OS 回收，分配方式类似于链表。参考如下代码：

```c
int main() {
	// C 中用 malloc() 函数申请
	char* p1 = (char *)malloc(10);
	cout<<(int*)p1<<endl;		//输出：00000000003BA0C0
	
	// 用 free() 函数释放
	free(p1);
   
	// C++ 中用 new 运算符申请
	char* p2 = new char[10];
	cout << (int*)p2 << endl;		//输出：00000000003BA0C0
	
	// 用 delete 运算符释放
	delete[] p2;
}
```

其中 p1 所指的 10 字节的内存空间与 p2 所指的 10 字节内存空间都是存在于堆。堆的内存地址生长方向与栈相反，由低到高，但需要注意的是，后申请的内存空间并不一定在先申请的内存空间的后面，即 p2 指向的地址并不一定大于 p1 所指向的内存地址，原因是先申请的内存空间一旦被释放，后申请的内存空间则会利用先前被释放的内存，从而导致先后分配的内存空间在地址上不存在先后关系。堆中存储的数据若未释放，则其生命周期等同于程序的生命周期。

关于堆上内存空间的分配过程，首先应该知道操作系统有一个记录空闲内存地址的链表，当系统收到程序的申请时，会遍历该链表，寻找第一个空间大于所申请空间的堆节点，然后将该节点从空闲节点链表中删除，并将该节点的空间分配给程序。另外，对于大多数系统，会在这块内存空间中的首地址处记录本次分配的大小，这样，代码中的delete语句才能正确地释放本内存空间。由于找到的堆节点的大小不一定正好等于申请的大小，系统会自动地将多余的那部分重新放入空闲链表。
#### 二者主要区别

##### 管理方式不同

栈由操作系统自动分配释放，无需我们手动控制；堆的申请和释放工作由程序员控制，容易产生内存泄漏

##### 空间大小不同

每个进程拥有的栈大小要远远小于堆大小。理论上，进程可申请的堆大小为虚拟内存大小，进程栈的大小 64bits 的 Windows 默认 1MB，64bits 的 Linux 默认 10MB

##### 生长方向不同

堆的生长方向向上，内存地址由低到高；栈的生长方向向下，内存地址由高到低

##### 分配方式不同

堆都是动态分配的，没有静态分配的堆。栈有 2 种分配方式：静态分配和动态分配。静态分配是由操作系统完成的，比如局部变量的分配。动态分配由`alloca()`函数分配，但是栈的动态分配和堆是不同的，它的动态分配是由操作系统进行释放，无需我们手工实现。

##### 分配效率不同

栈由操作系统自动分配，会在硬件层级对栈提供支持：分配专门的寄存器存放栈的地址，压栈出栈都有专门的指令执行，这就决定了栈的效率比较高。堆则是由C/C++提供的库函数或运算符来完成申请与管理，实现机制较为复杂，频繁的内存申请容易产生内存碎片。显然，堆的效率比栈要低得多。

##### 存放内容不同

栈存放的内容，函数返回地址、相关参数、局部变量和寄存器内容等。当主函数调用另外一个函数的时候，要对当前函数执行断点进行保存，需要使用栈来实现，首先入栈的是主函数下一条语句的地址，即扩展指针寄存器的内容（EIP），然后是当前栈帧的底部地址，即扩展基址指针寄存器内容（EBP），再然后是被调函数的实参等，一般情况下是按照从右向左的顺序入栈，之后是被调函数的局部变量，注意静态变量是存放在数据段或者 BSS 段，是不入栈的。出栈的顺序正好相反，最终栈顶指向主函数下一条语句的地址，主程序又从该地址开始执行。堆，一般情况堆顶使用一个字节的空间来存放堆的大小，而堆中具体存放内容是由程序员来填充的。

> [!NOTE]
>
> 无论是堆还是栈，在内存使用时都要防止非法越界，越界导致的非法内存访问可能会摧毁程序的堆、栈数据

#### 嵌入式开发中的栈

栈是一段连续的存储器空间，后放进去的先被取出，先放进去的后被取出(后来居上，跟向瓶子里放石子一样)，瓶底是高地址，瓶口是低地址。有一个堆栈指针指向正在放的位置。目的是保存现场，常用于中断

堆是全局的空间，空间占用不连续。

注意堆栈溢出

### 内存保护单元介绍

内存保护单元（memory protection unit），简称：MPU。

MPU的功能：

- 设置不同存储区域的存储器访问权限（特权级、用户级）

- 设置存储器(内存和外设)属性（可缓存、可缓冲、可共享）

相当于内存管理员，管理权限 & 通行规则/路径

好处：

- 阻止用户应用程序破坏操作系统使用的数据
- 阻止一个任务访问其他任务的数据区，从而隔离任务
- 把关键数据区域设置为只读，从根本上解决被破坏的可能
- 检测意外的存储访问，如堆栈溢出、数组越界等
- 将SRAM或RAM空间定义为不可执行，防止代码注入攻击

总之，提高嵌入式系统的健壮性，使系统更加安全

MPU可配置保护8/16个内存区域，每个区域最小要求256字节，且每个区域还可配置为8个子区域（大小一样）。

### Cache简介

Cache(高级缓存)是提升STM32性能的关键一步。

M7内核芯片做了一级Cache支持，Cache分为数据缓存D-Cache和指令缓存I-Cache。数据缓存D-Cache是解决CPU加速访问SRAM

![image-20250714140701614](C:\Users\xiaom\AppData\Roaming\Typora\typora-user-images\image-20250714140701614.png)

Cache支持4种基本操作：1,使能；2,禁止； 3,清空； 4,无效化。后两个是数据出现问题的操作

![image-20250714140936174](C:\Users\xiaom\AppData\Roaming\Typora\typora-user-images\image-20250714140936174.png)

保证cache有足够高的命中率，尽量少的cache miss，读/写速度会有比较大的提高。

#### Cache读操作

CPU要读取的SRAM区数据在Cache中已经加载好，叫读命中(Cache hit)； 如果Cache里面没有，这就是读Cache Miss。

![image-20250714141205460](C:\Users\xiaom\AppData\Roaming\Typora\typora-user-images\image-20250714141205460.png)

- Cache hit：直接从cache中读出数据即可

- Cache miss：两种处理方式

    - read through：直接从内存中读出，不用cache

    - read allocate：把数据从内存加载到cache，再从cache读取

#### Cache写操作

CPU要写的SRAM区数据在Cache中已经开辟了对应的区域，这就叫写命中(Cache hit)；如果Cache里面没有开辟对应的区域就是写Cache Miss。

![image-20250714141240747](C:\Users\xiaom\AppData\Roaming\Typora\typora-user-images\image-20250714141240747.png)

- Cache hit两种处理方式：

    - write through：直接写到内存中并同时放到cache里面，内存和cache同步更新，速度优势无法体现

    - write back：数据更新时只写入cache，只在数据被替换出cache时，被修改的cache数据才会写入内存

- Cache miss两种处理方式：

    - write allocate：先把要写的数据载入到cache，对cache写后，更新到内存

    - no write allocate：直接写入内存，不用cache

#### 数据不一致问题解决

- 设置共享属性，Cache相当于没有开启

- 软件进行cache维护

    - clean 清空：Cache已变化，SRAM数据未更新，DMA搬运数据前，将cache相对应数据更新到SRAM.

    - invalidate 无效化：SRAM数据已变化，Cache未更新，DMA搬运数据后，cache数据无效，需从SRAM中获取

## I2C

### 基本介绍

同步通信，两条线。SDA用来传递数据，SCL提供同步时钟脉冲，半双工通信，采用主从模式，只能由主机先发起通信，从机才能回复。总线上每个设备都有对应地址，主机发送数据最开始发送对应设备的地址。

主机的时钟信号作为所有设备的时钟源，SDA和SCL均有上拉电阻，未开始通信时，二者处于高电平，主机在时钟线高电平时拉低数据线，此时从机准备接收命令，开始正式I2C通信，主机在时钟线产生恒定频率时钟脉冲信号，时钟线在低电平时，主机设置数据线的电平；时钟线在高电平时，从机读取主机刚设置的数据线的电平。主机发完一字节，从机需要发一个ACK应答信号，即在时钟线低电平时，由从机拉低数据线。主机在时钟线高电平时拉高数据线，宣告通信完成。

### 轮询模式

> [!NOTE]
>
> 只有在主机设置开始和结束时能在时钟线高电平时操作数据线，否则只能在时钟线低电平操作数据线

## ADC

模拟-数字转换技术(Analog to Digital Conversion)

### 工作原理

逐次逼近法(SAR)：通过不断二分比较最终确定电压值，内置12位ADC，0-4095(0-3.3V)，线性对应

测量过程：通过GPIO把待测电压采集到电容，切断与待测信号的链接来把待测信号保持在电容上从而防止待测信号波动影响结果。然后用VCC/2(011111111111)与待测电压进行比较，确定第12位的值，然后用第12位是已确定的值，第11位是1确定第11位的值，按照这样进行二分比较，比较之后把结果放入ADC转换结果寄存器，程序从此获得结果，/4095*3.3V即可获得结果

### 采样通道

- 外部通道：ADC外部通道1-10
- 内部通道
    - 温度传感器
    - 内部参考电压

### ADC结构

两个ADC:ADC1,ADC2

- 注入组
- 规则组(普通组)：把某个ADC通道注册在上面，触发ADC时会对该通道进行转换，结果放入规则通道数据寄存器，等待被读取。也可以注册多个通道，让ADC依次转换

> [!NOTE]
>
> ADC时钟频率不宜过快，时钟树可进行配置ADC的分频

> [!TIP]
>
> 上电之后进行ADC校准，调用HAL_ADCEx_Calibration_Start(&hadc1)即可
>

## RTC

### RTC简介

RTC=Real Time Clock实时时钟

简化版定时器，核心是计数器，32位0-4294967295，还有预分频器做到每1s增加1

UNIX时间戳：1970.1.1 00:00:00到当前时刻的秒数

RTC无符号位，可记到2106年

RTC掉电继续走时：为VBAT引脚供电，STM32会维持后备区域的运行，该区域耗电量少，板载一颗纽扣电池即可

时钟源

- LSE：低速外部时钟，
- LSI：低速内部时钟
- HSE128分频：高速外部时钟

> [!NOTE]
>
> 只有LSE可以在VDD掉电后继续提供时钟信号，32.768kHz，经过预分频器变成1Hz

后备区域还有备份寄存器，做到掉电不丢失，还有入侵保护(RTC-Tamper设置),可放置激活码

HAL_RTC_SetTime函数不涉及日期，日起在HAL_RTC_SetDate函数

#include "myrtc.h"

// RTC已经被初始化的值 记录在RTC_BKP_DR1中
#define RTC_INIT_FLAG 0x2333

/**
 * @brief  进入RTC初始化模式
 * @param  hrtc  指向包含RTC配置信息的RTC_HandleTypeDef结构体的指针
 * @retval HAL status
 */
static HAL_StatusTypeDef RTC_EnterInitMode(RTC_HandleTypeDef *hrtc)
{
    uint32_t tickstart = 0U;

    tickstart = HAL_GetTick();
    /* 等待RTC处于INIT状态，如果到达Time out 则退出 */
    while ((hrtc->Instance->CRL & RTC_CRL_RTOFF) == (uint32_t)RESET)
    {
        if ((HAL_GetTick() - tickstart) > RTC_TIMEOUT_VALUE)
        {
            return HAL_TIMEOUT;
        }
    }

    /* 禁用RTC寄存器的写保护 */
    __HAL_RTC_WRITEPROTECTION_DISABLE(hrtc);

    return HAL_OK;
}

/**
 * @brief  退出RTC初始化模式
 * @param  hrtc   指向包含RTC配置信息的RTC_HandleTypeDef结构体的指针
 * @retval HAL status
 */
static HAL_StatusTypeDef RTC_ExitInitMode(RTC_HandleTypeDef *hrtc)
{
    uint32_t tickstart = 0U;

    /* 禁用RTC寄存器的写保护。 */
    __HAL_RTC_WRITEPROTECTION_ENABLE(hrtc);

    tickstart = HAL_GetTick();
    /* 等到RTC处于INIT状态，如果到达Time out 则退出 */
    while ((hrtc->Instance->CRL & RTC_CRL_RTOFF) == (uint32_t)RESET)
    {
        if ((HAL_GetTick() - tickstart) > RTC_TIMEOUT_VALUE)
        {
            return HAL_TIMEOUT;
        }
    }

    return HAL_OK;
}

/**
 * @brief  写入RTC_CNT寄存器中的时间计数器。
 * @param  hrtc   指向包含RTC配置信息的RTC_HandleTypeDef结构体的指针。
 * @param  TimeCounter: 写入RTC_CNT寄存器的计数器
 * @retval HAL status
 */
static HAL_StatusTypeDef RTC_WriteTimeCounter(RTC_HandleTypeDef *hrtc, uint32_t TimeCounter)
{
    HAL_StatusTypeDef status = HAL_OK;

    /* 进入RTC初始化模式 */
    if (RTC_EnterInitMode(hrtc) != HAL_OK)
    {
        status = HAL_ERROR;
    }
    else
    {
        /* 设置RTC计数器高位寄存器 */
        WRITE_REG(hrtc->Instance->CNTH, (TimeCounter >> 16U));
        /* 设置RTC计数器低位寄存器 */
        WRITE_REG(hrtc->Instance->CNTL, (TimeCounter & RTC_CNTL_RTC_CNT));

        /* 退出RTC初始化模式 */
        if (RTC_ExitInitMode(hrtc) != HAL_OK)
        {
            status = HAL_ERROR;
        }
    }

    return status;
}

/**
 * @brief  读取RTC_CNT寄存器中的时间计数器。
 * @param  hrtc   指向包含RTC配置信息的RTC_HandleTypeDef结构体的指针。
 * @retval 时间计数器
 */
static uint32_t RTC_ReadTimeCounter(RTC_HandleTypeDef *hrtc)
{
    uint16_t high1 = 0U, high2 = 0U, low = 0U;
    uint32_t timecounter = 0U;

    high1 = READ_REG(hrtc->Instance->CNTH & RTC_CNTH_RTC_CNT);
    low = READ_REG(hrtc->Instance->CNTL & RTC_CNTL_RTC_CNT);
    high2 = READ_REG(hrtc->Instance->CNTH & RTC_CNTH_RTC_CNT);

    if (high1 != high2)
    {
        /* 当读取CNTL和CNTH寄存器期间计数器溢出时, 重新读取CNTL寄存器然后返回计数器值 */
        timecounter = (((uint32_t)high2 << 16U) | READ_REG(hrtc->Instance->CNTL & RTC_CNTL_RTC_CNT));
    }
    else
    {
        /* 当读取CNTL和CNTH寄存器期间没有计数器溢出, 计数器值等于第一次读取的CNTL和CNTH值 */
        timecounter = (((uint32_t)high1 << 16U) | low);
    }

    return timecounter;
}

/**
 * @brief 设置RTC时间
 * @param time 时间
 * @retval HAL status
 */
HAL_StatusTypeDef myRTC_SetTime(struct tm *time)/*tm是time.h里面表示日期时间的结构体*/

 * @retval 时间
 */
struct tm *myRTC_GetTime()
{
    time_t unixTime = RTC_ReadTimeCounter(&hrtc);/*time_t是64位整数*/
    return gmtime(&unixTime);
}

## DMA

### DMA简介

DMA=Direct Memory Access

有DMA控制器，每个DMA控制器分管几条通道

DMA1管7条，DMA2管5条(大容量的才有DMA2)

每个DMA通道都有对应的外设地址寄存器和存储器地址寄存器，还有一些存在其他寄存器上的设置信息(传输方向、数据宽度、地址自增、循环模式、优先级......)

每次搬运数据都需要触发信号，如串口发送完毕。但是这个触发信号与通道绑定，UART2的触发信号只链接在DMA1的通道7上，只有它可以相应信号

> [!NOTE]
>
> 一个通道只能被一个外设使用，所以要进行取舍

同一时刻只能对一个通道进行搬运，可以配置优先级，两个DMA搬运请求同时到达时，DMA控制器会先搬运优先级高的通道

### WS2812控制RGB小灯

一个RGB小灯需要3路PWM，需要借助控制芯片，如WS2812，可以发出三路PWM，单片机告知WS2812每颗小灯的亮度即可

RGB表示法

0->0xFF:灭->亮（一字节）

一个颜色3字节即可表示

引脚：

- VDD
- GND
- DI(Data Input)：接收到Reset信号之后，显示在灯上
- DO(Data Output)：用来级联，发给下一个WS2812，Reset是同时收到的

使用归零码：一个周期内，用二进制传输数据位，在数据位脉冲结束后，需要维持一段时间的低电平(通过零电平隔开每位数据)，通过每位数据中高电平时间的长和短表示0和1![4657880e83e01a44f3bde3691602a6ea](D:\xwechat_files\wxid_85qaa11voxkj32_0996\temp\RWTemp\2025-07\9e20f478899dc29eb19741386f9343c8\4657880e83e01a44f3bde3691602a6ea.jpg)

规格书说Reset是大于125$\mu s$的持续低电平

信号频率800Hz，占空比2/3是高电平，1/3是低电平

如何发送？延时和控制GPIO高低电平，占用CPU

可以用PWM，调整比较寄存器使用DMA

定时器有一个可以触发DMA的事件：输出比较，计数器的值=输出比较寄存器时，会触发DMA搬运，把下个周期所用数据搬运到输出比较寄存器，而本周期不受影响，所以准备一个储存输出比较寄存器值的数组即可

DMA Request设置为TIM3_CH1/TRIG(触发事件，包括计数器启动、停止、初始化)而不是TIM3_CH4/UP(更新事件，计数器重装载时)

> [!NOTE]
>
> 输入捕获寄存器和输出比较寄存器是同一个寄存器

### 播放音乐

```c
typedef struct {
  uint16_t frequency;
  float period;
} Bate;
TIM_HandleTypeDef htim4;
const Bate MoChouXiang[] = {
    // 我被困在了
    {M6, 1},
    {M5, 1},
    {M3, 1},
    {M5, 0.5f},
    {M5, 0.5f},
    ......
}
int main(void) {
  HAL_Init();
  SystemClock_Config();
  MX_GPIO_Init();
  MX_TIM4_Init();
  // 开始PWM输出
  HAL_TIM_PWM_Start(&htim4, TIM_CHANNEL_4);
  // TIM4的计数频率
  uint32_t timFrequency = TIM_GetCounterFreq(&htim4);
  // 播放状态
  uint8_t playState = 0;
  // 播放进度
  uint32_t playIndex = 0;
  // 节拍速度(每分钟多少拍)
  uint8_t bpm = 132;
  // 每拍的持续时间
  float noteDuration = 1000 * 60 / bpm;
  while (1) {
    // 按键检测，切换播放与暂停
    if (HAL_GPIO_ReadPin(KEY1_GPIO_Port, KEY1_Pin) == GPIO_PIN_RESET) {
      HAL_Delay(10);
      if (HAL_GPIO_ReadPin(KEY1_GPIO_Port, KEY1_Pin) == GPIO_PIN_RESET) {
        playState = !playState;
        while (HAL_GPIO_ReadPin(KEY1_GPIO_Port, KEY1_Pin) == GPIO_PIN_RESET)
          ;
      }
    }
    // 播放
    if (playState) {
      const Bate bate = MoChouXiang[playIndex];
      if (bate.frequency == P0) {
        // 休止符
        __HAL_TIM_SET_COMPARE(&htim4, TIM_CHANNEL_4, 0);
      } else {
        // 将频率转换为计数值, 设置到自动重装载寄存器
        uint32_t arr = timFrequency / bate.frequency;
        __HAL_TIM_SET_AUTORELOAD(&htim4, arr);
        // 设置占空比为20%
        __HAL_TIM_SET_COMPARE(&htim4, TIM_CHANNEL_4, arr / 5); // 20%占空比
        // 从0开始计数 重置PWM波形
        __HAL_TIM_SetCounter(&htim4, 0);
      }
      // 延时该音符的持续时间 (5ms的空白以区分连续两个相同的音符)
      HAL_Delay((uint32_t)(bate.period * noteDuration) - 5);
      __HAL_TIM_SET_COMPARE(&htim4, TIM_CHANNEL_4, 0);
      HAL_Delay(5);

      // 下一个音符
      playIndex++;
      // 播放结束
      if (playIndex >= sizeof(MoChouXiang)) {
        playState = 0;
        playIndex = 0;
      }
    } else {
      __HAL_TIM_SET_COMPARE(&htim4, TIM_CHANNEL_4, 0);
    }
  }
}

#### 基本信息

在简谱中的 4/4 下面的4表示以四分音符为一个拍，上面的4表示每小节4拍。

然后bmp是指曲子的速度，单位是每分钟多少拍，这里的132就表示每分钟132拍。

我们的程序中就有`bmp`变量，可以通过修改这个变量来调整曲子的速度。

#### 高低中音

一半的数字我们都认为是中音，1 2 3使用M1 M2 M3宏定义的频率即可，如若数字下有·，则表示低音，我们使用L1 L2 L3宏定义的频率， 若数字上有·，则表示高音，我们使用H1 H2 H3宏定义的频率。

#### 音符的持续时间

在简谱中，一个普通的，下面没有带横线的数字，就是一个四分音符，持续1拍的时间。

例如谱子刚上来的 6 5 3 就是3个四分音符，分别是6、5、3，每个音符持续1拍。所以转换到程序中就是：

```c
    {M6, 1}, {M5, 1}, {M3, 1},
```



而如果数字下面有一根横线，就表示这个音符是一个八分音符，持续的时间便是四分音符的一半，也就是0.5拍。

例如第一句的后面的 5 5 就是两个八分音符，持续0.5拍，所以转换到程序中就是：

```c
    {M5, 0.5f}, {M5, 0.5f},
```



而假若数字下面带有两个横线，就表示这个音符是一个十六分音符，持续的时间便是四分音符的四分之一，也就是0.25拍。 同理，若数字下有三个横线，则表示三十二分音符，持续的时间便是四分音符的八分之一，也就是0.125拍。

例如简谱中“呼吸声”三个字对应的 1 3 5，其中1 3下有三个横线 5下有两个横线，转换到程序中就是：

```c
    {M1, 0.125f}, {M3, 0.125f}, {M5, 0.25f},
```



假若数字后有•，则表示这个音符延长半拍，也就是四分音符的1.5倍时间。例如“不能理顺我自己的疑问”中的“疑”对应的音符5•，转换到程序中就是：

```c
    {M5, 1.5f},
```



而若数字后有横杠-，则表示音频延长一拍，也就是四分音符的2倍时间。而且延音符号可以叠加，例如数字后面若有--•，则表示延长2.5拍。

### RTC相关寄存器

#### 后备寄存器和RTC寄存器特性（F1）

- 部分寄存器写保护：RTC_PRL、RTC_ALR、RTC_CNT和RTC_DIV寄存器不会被系统复位 
- 数据存储功能：RTC和后备寄存器不会被系统或电源复位源复位；当从待机模式唤醒时，也不会被复位。 后备寄存器可用于保存掉电时的数据。
- 独立工作：RTC和后备寄存器通过一个开关供电，在VDD有效时该开关选择VDD供电，否则由VBAT引脚供电。在VBAT供电时仍可继续工作。
- 2个独立复位：APB1接口由系统复位；RTC核心只能由后备域复位

#### F1 RTC相关寄存器介绍

##### 基本配置步骤

- 使能对RTC的访问：使能PWR && BKP时钟、使能对后备寄存器和RTC的访问权限
- 设置RTC时钟源：激活LSE，设置RTC的计数时钟源为LSE
- 进入配置模式：等待 RTOFF位为1，设置CNF位为1	
- 设置RTC寄存器：设置分频值、计数值等，一般先只设置分频值，CNT的设置独立
- 退出配置模式：清除CNF位，等待RTOFF 为1即配置完成
