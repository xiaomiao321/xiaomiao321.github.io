轻量级音乐播放器 MusicMenuLite

Created2026-03-14|Updated2026-03-14|TutorialEmbedded
|Word Count:737|Reading Time:2mins|Post Views:

一、引言

MusicMenuLite 是一个轻量级的音乐播放器，作为 Buzzer 模块的替代或补充。它旨在提供一个功能完整但界面更简洁、信息密度更高的播放体验。

二、实现思路

2.1 共享与复用

本模块高效地复用了 Buzzer 模块中已存在的资源。通过 #include "Buzzer.h"，直接访问已定义在 PROGMEM 中的 songs 数组、歌曲数量 numSongs 以及播放模式 PlayMode 枚举。这种做法避免了重复定义大量数据，节省了宝贵的程序存储空间。

2.2 独立的播放任务

本模块拥有自己独立的后台播放任务 MusicLite_Playback_Task，负责根据选定的播放模式来解码并播放 PROGMEM 中的歌曲数据。它支持暂停/恢复功能，并通过一系列 volatile 修饰的全局共享变量与 UI 线程同步状态。

2.3 信息化的 UI 设计

MusicMenuLite 的 UI 设计哲学是”信息优先”：

  • 多行文本信息：清晰展示歌曲名、当前音符频率和时长、播放模式、系统实时时间、播放/暂停状态、数字格式的播放进度
  • 彩色进度条：进度条颜色根据歌曲索引从预定义颜色数组中选取
  • 高频刷新：播放界面刷新频率为每秒 5 次（200ms 间隔）

三、核心代码

状态同步的共享变量

static volatile int shared_song_index = 0;
static volatile int shared_note_index = 0;
static volatile int shared_total_notes = 0;
static volatile PlayMode shared_play_mode = LIST_LOOP;
static volatile TickType_t current_note_start_tick = 0;
static volatile int shared_current_note_duration = 0;
static volatile int shared_current_note_frequency = 0;

后台播放任务

void MusicLite_Playback_Task(void *pvParameters) {
    int songIndex = *(int*)pvParameters;
    
    for (;;) {
        Song song;
        memcpy_P(&song, &songs[songIndex], sizeof(Song));
        
        for (int i = 0; i < song.length; i++) {
            while (isPaused) { vTaskDelay(pdMS_TO_TICKS(50)); }
            
            int note = pgm_read_word(song.melody + i);
            int duration = pgm_read_word(song.durations + i);
            
            // 更新共享变量
            shared_note_index = i;
            shared_current_note_frequency = note;
            shared_current_note_duration = duration;
            current_note_start_tick = xTaskGetTickCount();
            
            if (note > 0) {
                tone(BUZZER_PIN, note, duration * 0.9);
            }
            vTaskDelay(pdMS_TO_TICKS(duration));
        }
        
        // 根据播放模式选择下一首
        if (currentPlayMode == LIST_LOOP) {
            songIndex = (songIndex + 1) % numSongs;
        }
    }
}

精确的已播放时间计算

static uint32_t calculateElapsedTime_ms() {
    uint32_t elapsed_ms = 0;
    
    // 累加已播放完毕的音符时长
    for (int i = 0; i < shared_note_index; i++) {
        elapsed_ms += pgm_read_word(song.durations + i);
    }
    
    // 加上当前正在播放的音符已经历的时间
    TickType_t current_note_elapsed_ticks = 
        xTaskGetTickCount() - current_note_start_tick;
    elapsed_ms += (current_note_elapsed_ticks * 1000) / configTICK_RATE_HZ;
    
    return elapsed_ms;
}

播放界面绘制

void displayPlayingScreen_Lite(uint16_t progress_bar_color) {
    menuSprite.fillSprite(TFT_BLACK);
    
    // 歌曲名
    menuSprite.drawString(songs[shared_song_index].name, 120, 20);
    
    // 音符信息
    snprintf(noteInfoStr, sizeof(noteInfoStr), "%d Hz  %d ms", 
             shared_current_note_frequency, shared_current_note_duration);
    menuSprite.drawString(noteInfoStr, 120, 50);
    
    // 播放模式
    menuSprite.drawString(playModeStr[shared_play_mode], 120, 80);
    
    // 系统时间
    strftime(timeStr, sizeof(timeStr), "%H:%M:%S", &timeinfo);
    menuSprite.drawString(timeStr, 120, 110);
    
    // 进度条
    uint32_t elapsed_ms = calculateElapsedTime_ms();
    uint32_t total_ms = calculateSongDuration_ms(shared_song_index);
    int progressWidth = (elapsed_ms * 220) / total_ms;
    menuSprite.fillRoundRect(10, 165, progressWidth, 10, 5, progress_bar_color);
    
    menuSprite.pushSprite(0, 0);
}

四、总结

MusicMenuLite 通过共享 Buzzer 模块的歌曲数据节省存储空间，独立播放任务保证 UI 流畅，精确的时间计算实现平滑进度条，信息化 UI 设计提供丰富的状态反馈。
