游戏模块 - 多合一迷你游戏启动器

Created2026-03-14|Updated2026-03-14|TutorialEmbedded
|Word Count:815|Reading Time:2mins|Post Views:

一、引言

本模块作为一个迷你游戏平台，集成了一个可视化的游戏选择菜单和多个独立的小游戏。用户可以通过横向滚动界面浏览所有可玩的游戏，并通过单击启动所选游戏。

二、实现思路

2.1 游戏启动器菜单

表驱动设计：与主菜单类似，GamesMenu 也采用”表驱动”设计。一个 GameItem 结构体数组集中管理了所有游戏，每个条目包含游戏的名称、图标和一个指向游戏主函数的函数指针。

struct GameItem {
    const char *name;
    const uint16_t *image;
    void (*function)(); // 指向游戏主函数的指针
};

const GameItem gameItems[] = {
    {"Conway's Game", bird_big, ConwayGame},
    {"Buzzer Tap", bird_big, BuzzerTapGame},
    {"Time Challenge", Timer, TimeChallengeGame},
    {"Flappy Bird", bird_big, flappy_bird_game},
};

滚动与动画：游戏菜单的 UI 和交互逻辑复刻了主菜单的设计，包括双缓冲渲染和平滑的横向滚动动画。

输入处理：旋转编码器用于选择游戏，单击用于启动游戏，双击用于退出整个游戏模块。

2.2 独立的游戏循环

每个游戏本身都是一个独立的函数，拥有自己的主循环 while(true)。当用户从菜单中选择一个游戏后，对应的函数被调用，程序流程就进入了该游戏的内部循环。

2.3 集成的游戏示例

康威生命游戏：经典的元胞自动机模拟，在二维网格上根据生存/死亡规则演化生命形态。

蜂鸣器反应力：考验反应速度的游戏，在提示音响起后的极短时间窗口内按下按钮才能得分。

时间挑战：系统给出随机目标时间，玩家尝试在最接近目标时间的时刻按下按钮。

Flappy Bird：经典小游戏复刻，通过单击让小鸟跳跃以穿过管道缝隙。

三、核心代码

函数指针作为游戏入口

struct GameItem {
    const char *name;
    const uint16_t *image;
    void (*function)();
};

// 启动游戏
if (gameItems[game_picture_flag].function) {
    gameItems[game_picture_flag].function();
}

区分单击与双击

// 检测双击
if (readButton()) {
    if (gamesDetectDoubleClick()) {
        return; // 退出游戏菜单
    } else {
        gamesMenuLastClickTime = millis();
        gamesMenuSingleClickPending = true;
    }
}

// 延迟确认单击
if (gamesMenuSingleClickPending && (millis() - gamesMenuLastClickTime > 500)) {
    gamesMenuSingleClickPending = false;
    // 启动游戏
    if (gameItems[game_picture_flag].function) {
        gameItems[game_picture_flag].function();
    }
}

Flappy Bird 游戏示例

void flappy_bird_game() {
    // 初始化游戏变量
    int bird_y = 120;
    int pipe_x = 240;
    int score = 0;
    bool game_over = false;

    while (true) {
        // 退出条件检查
        if (readButtonLongPress()) return;
        if (gamesDetectDoubleClick()) return;

        if (!game_over) {
            // 处理用户输入（跳跃）
            if (readButton()) {
                bird_velocity = -5;
            }
            
            // 更新小鸟位置（重力）
            bird_velocity += 0.3;
            bird_y += bird_velocity;
            
            // 更新管道位置
            pipe_x -= 2;
            
            // 碰撞检测
            if (bird_y < 0 || bird_y > 239 || 
                (pipe_x < 20 && bird_y < gap_top || bird_y > gap_bottom)) {
                game_over = true;
            }
        } else {
            // 处理游戏结束后重启
            if (readButton()) return;
        }

        // 绘制游戏画面
        menuSprite.fillSprite(TFT_BLACK);
        // 绘制小鸟、管道、分数等
        menuSprite.pushSprite(0, 0);

        vTaskDelay(pdMS_TO_TICKS(10));
    }
}

四、总结

游戏模块通过表驱动设计实现游戏的易扩展性，函数指针作为游戏入口实现解耦，独立的游戏循环保证每个游戏的完整性，单击/双击区分实现启动/退出功能。
