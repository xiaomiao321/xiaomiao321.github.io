电路结构和电路调试

缩写

  1. GND(Ground)：地，整个电路的参考点，决定了电压基准
  2. VCC(C=circuit)：接入板子正向电源的电压
  3. VDD(D=device)：集成 IC 的电压
  4. VEE(E=emitter)：在电路指代电源负极。直流电源分为单电源和双电源
    • 单电源：电压只有高于 GND 的部分
    • 双电源：既有正电压又有负电压
  5. VSS(S=series)：公共连接点，约等于 GND
  6. Vpp(p=peak)：电压峰峰值，最大值 - 最小值
  7. VBAT(BAT=BATTERY)：电源电压
  8. 网络：联通的所有节点

电路结构

IO 口

杜邦线与排针
  • 简单但是连接不可靠，且传输大电流发烫严重、损耗大
  • 只适用于简单的东西
SMA(同轴线缆)
  • 用于高频或对信号要求比较高的场景
FPC 连接器
  • 用于两块板子的连接，集成度高、简单、信号稳定性比较好
USB 接口

详细见 USB

供电管理

[!NOTE]

通常板子的供电和芯片所需电压不一样

LDO(Low Dropout Regulator)

LDO 是低压差线性稳压器

  • 优势：伺服电路结构简单、输出电压较稳定
  • 劣势：效率低，发热量大（压差损耗功率全部在 LDO 芯片上转化为热量），能够承载的功率低，一般只做降压电路
DC-DC 转换器（直流——直流）
  • 原理：通过快速开启和关闭开关元件（MOSFET 晶体管）输出 PWM 波来调节电压或电流的平均值，从而达到改变输出电压的目的。
  • 优势：效率高，能承载较大电路
  • 劣势：
    1. 电路结构复杂
    2. 有开关高频信号，干扰信号传输

电流较大一般用 DC-DC

电路调试

万用表

蜂鸣档在短路的时候会响

信号发生器

学生电源

第一次上电把限流调的低一些，防止由于短路烧毁

示波器

调试之前先确认电气连接，先补锡，再测试

先检查是否短路，蜂鸣档测 VCC 和 GND，显示 OL 说明没短路

测量时保证不要同时碰到芯片的两个角

触发电平在波形范围内才能让波形稳定显示

GPIO

General Purpose Input Output

输出

推挽输出

两个 MOS 管：强上拉强下拉，在高低两种电平下都具有驱动 (输出电流) 能力。但是驱动能力受芯片输出能力的限制

开漏输出

一个 MOS 管：强下拉，外部需要接上拉电阻，高电平输出能力由 VCC 提供，输出能力强

输入

上拉输入

输入接上拉电阻，悬空时输入高电平，防止悬空时电压不可控（有效信号是低电平，常态高电平）

下拉输入

与上拉输入相反（有效信号是高电平，常态低电平）

例如按键有效电平是低电平，应该采用上拉输入

USB 和电源简介

USB

全称：通用串行总线（Universal Serial Bus)

  • 串行：所有信号从一根线传输按照时间先后顺序传输数据，稳定性比并行的强一些
  • 总线：所有设备挂在总线上，所有设备都可以从总线上读取信息，可以减少对引脚资源的占用

标准 USB type-A(2.0)

四根线：

  • VCC
  • GND
  • DP(数据正线)
  • DN(数据负线)

差模信号的电压取决于 DP 和 DN 之间的电压（差分传输），防止环境噪声干扰。差分走线必须等长、等宽、紧密靠近、在同一层面。

USB-C

雷电标志：有这个标志的速率快很多

USB-C 引脚定义

由于对称（除了 CC1、CC2）可以正负反着插

  • D+ 和 D-是差分信号线，需要等长、等宽，在同一面板上
  • RX、TX 是串行总线的信号线
  • VBUS(Voltage Bus) 是总线电压，为 USB-C 供电，可以加一个 1uF 电容接地进行输入电压滤波
  • CC1、CC2：PD 充电协议握手线
    • 确定是同向插进去还是反向插进去
    • 确定供电和收电端
    • 确定从机和主机能接受的电压

电源

LDO

工作原理

若比基准电压高，MOS 关断一点，反之打开一点

由基准电压和电阻分压公式可计算 $V_{out}$

电容是为了保持电源的相对稳定

耗散电流和负载要的电流是一样的

耗散功率 = $|V_{OUT}-V_{IN}| \cdot I_{LOAD}$

常用的：

  • AMS1117-3.3/AMS1117-5.0：输出电压不可调
  • AMS1117-ADJ：输出电压可调

DC-DC

类似于 PWM，MOS 要么完全导通（MOS 上电压=0），要么完全截止（MOS 上电流=0），这两种状态 MOS 上都不会有功耗，效率高

常用 TPS5430

应用举例：给 STM32 供电

用 TPS5430 把 12V 降到 5V，再用 AMS1117 把 5V 降到 3.3V

原因：

  • 直接用 AMS1117：功率过高，发热严重
  • 直接用 TPS5430：会有波动导致工作不稳定

MCU 及其伺服电路、通信协议基础

MCU 及其伺服电路

浅黄色的是供电脚：

  • VDDA：模拟（连续值而非离散值）电源

灰色的是 IO 口

绿色的决定芯片的工作状态：

  • NRST：复位引脚，用于重启单片机（一般低电平重启）

C3 电容是为了让上电的时候 NRST 保持低电平使单片机重启，不会从中间任何地方运行，之后充电单片机就正常运行（缓启动）

  • BOOT0

SWD(Serial Wire Debug)：

  • SWDIO(双向数据线)
  • SWCLK(时钟线)

通信协议基础

串口

通用异步收发器（Universal Asynchronous Receiver/Transmitter)，通常称作 UART，是一种串行、异步、全双工的通信协议。

  • 异步通信以一个字符为传输单位，通信中两个字符间的时间间隔多少是不固定的，然而在同一个字符中的两个相邻位间的时间间隔是固定的。
  • 数据传送速率用波特率来表示，即每秒钟传送码元数量。
报文格式

串口的报文数据由：起始位（1bit）+ 数据位（58bit）+ 奇偶校验位（1bit）+ 停止位（11.5bit）

  • 起始位：在发送有效数据前，无需配置，会自动产生 1bit 逻辑”0”的低电平的起始位,表示串口数据传输开始
  • 数据位：决定了通信过程中传输的有效数据位数，数据位通常有 5、6、7、8 bit
  • 校验位：校验方式有奇校验、偶校验、0 校验和 1 校验
  • 停止位：可以是 1bit、1.5bit 或者 2bit 逻辑”1” 高电平
  • 空闲位：空闲位不算是串口报文内的数据，它是发送完一组报文后，总线会自动将电平拉高

[!NOTE]

接线的时候 TX 和 RX 要对着接

RS485 与 RS232

二者都是串口的衍生

  • RS232：电平转换芯片把 TTL 电平转化为 232 电平 (高电平 3V 到 15V，低电平 -3V 到 -15V) 增强抗干扰能力，传输距离更长
  • RS485：电平转换芯片把输入的串口信号转化为差分信号，A>B 表示 0，反之表示 1，抗干扰能力强。且一般是半双工，可以进行一主多从的和很多设备进行通讯

CAN 总线

多用于汽车，CAN=Controller Area Network，控制器局域网

单片机信号经过 CAN 收发芯片转化为差分信号

数据帧
  • 起始位：必须是逻辑 0
  • 识别码：11 位，表明发给哪个设备，也代表了优先级
  • RTR 位：用来区分数据帧和远程请求帧
  • 控制码：6 位，控制数据长度
  • CRC 码：16 位，循环冗余校验位
  • ACK 码：确认应答
  • 结束位：7 位 1

I2C

Inter-Integrated Circuit

一主多从，两根线，时钟线 SCL 和数据线 SDA，两根线都要加上拉电阻，空闲状态二者都是高电平

数据帧
  • 起始位：SCL 为高电平时，SDA 由高到低（下降沿）
  • 7 位设备地址码
  • 读/写位：写=0，读=1
  • 应答信号：从机发送给主机
  • 八位寄存器地址
  • 应答信号
  • 八位给寄存器写入的数据
  • 应答信号
  • 停止位：SCL 为高时，SDA 由低到高

SPI

Serial Peripheral Interface：串行外设接口，高速、全双工

一主多从，四根线：

  • 片选 SS：确定哪一个芯片通信，大多数低电平有效
  • SCK：主设备产生时钟信号线，有空闲高/低，时钟上升/下降沿采集，共 4 种模式
  • MOSI：主设备输出，从设备输入
  • MISO：主设备输入，从设备输出

同步：依赖于时钟

异步：不依赖于时钟，而是依赖于二者约定好的频率
