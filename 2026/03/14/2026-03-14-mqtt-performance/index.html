一、MQTT 模块

1.1 引言

MQTT（Message Queuing Telemetry Transport）是一种轻量级的物联网消息传递协议。本模块通过集成 PubSubClient 库，使多功能时钟能够连接到公网 MQTT 服务器（巴法云），实现远程控制。

1.2 实现思路

连接管理：

  • setupMQTT() 初始化 MQTT 连接，配置服务器地址、端口和回调函数
  • connectMQTT() 可视化连接过程，输出到 TFT 屏幕
  • loopMQTT() 持续检查连接状态，断开时自动重连

指令处理与函数指针：

  • 回调函数 callback 处理收到的消息
  • 指令匹配 menuItems 数组中的菜单项名称
  • 全局函数指针 requestedMenuAction 实现主循环与回调的解耦

1.3 核心代码

#include <PubSubClient.h>

WiFiClient espClient;
PubSubClient client(espClient);

volatile void (*requestedMenuAction)() = nullptr;
volatile bool exitSubMenu = false;

// MQTT 消息回调
void callback(char* topic, byte* payload, unsigned int length) {
    payload[length] = '\0';
    String message = String((char*)payload);
    
    if (message.equalsIgnoreCase("exit")) {
        exitSubMenu = true;
        return;
    }
    
    // 遍历菜单项匹配指令
    for (int i = 0; i < MENU_ITEM_COUNT; i++) {
        if (message.equalsIgnoreCase(menuItems[i].name)) {
            requestedMenuAction = menuItems[i].action;
            tone(BUZZER_PIN, 1500, 100);
            break;
        }
    }
}

void setupMQTT() {
    if (WiFi.status() == WL_CONNECTED) {
        client.setServer(MQTT_SERVER, MQTT_PORT);
        client.setCallback(callback);
        connectMQTT();
    }
}

void loopMQTT() {
    if (!client.connected()) {
        // 尝试重连
    }
    client.loop();
}

// 在主循环中
if (requestedMenuAction != nullptr) {
    requestedMenuAction();
    requestedMenuAction = nullptr;
    showMenuConfig();
}

1.4 volatile 关键字的重要性

extern volatile void (*requestedMenuAction)();
extern volatile bool exitSubMenu;

volatile 确保每次访问变量时都从主内存读取最新值，防止编译器优化导致的问题。

二、PC 性能监控模块

2.1 引言

本模块将 ESP32 设备变成外置 PC 性能监控面板，通过串口与 PC 上位机通信，接收 CPU、GPU、RAM 性能数据并可视化展示。

2.2 实现思路

多任务并行处理：

  1. SERIAL_Task：串口接收任务，监听串口并将数据存入缓冲区
  2. Performance_Task：UI 刷新任务，定期更新屏幕显示
  3. Performance_Init_Task：一次性初始化任务，绘制静态元素

数据流与解析：

  • 接收特定格式字符串如 "C 55c 30% G 65c 80% RL 45.5%|..."
  • parsePCData() 函数暴力解析字符串提取数据
  • 互斥锁 xPCDataMutex 保护共享数据

数据可视化：

  • 数字面板显示当前性能数据
  • 多轨折线图绘制变化趋势

2.3 核心代码

struct PCData {
    float cpu_temp, cpu_load;
    float gpu_temp, gpu_load;
    float ram_load;
    char cpu_model[50], gpu_model[50];
    bool valid;
};

struct PCData pcData;
SemaphoreHandle_t xPCDataMutex = NULL;

// 串口接收任务
void SERIAL_Task(void *pvParameters) {
  for (;;) {
    if (Serial.available()) {
      char inChar = (char)Serial.read();
      if (bufferIndex < BUFFER_SIZE - 1) {
        inputBuffer[bufferIndex++] = inChar;
        inputBuffer[bufferIndex] = '\0';
      }
      if (inChar == '\n') {
        stringComplete = true;
      }
    }
    if (stringComplete) {
      parsePCData();
      stringComplete = false;
      resetBuffer();
    }
    vTaskDelay(pdMS_TO_TICKS(10));
  }
}

// 数据解析
void parsePCData() {
  struct PCData newData;
  // 使用 strstr, strchr 等解析 inputBuffer
  
  // 获取互斥锁后更新
  if (xSemaphoreTake(xPCDataMutex, portMAX_DELAY) == pdTRUE) {
    pcData = newData;
    xSemaphoreGive(xPCDataMutex);
  }
}

// FreeRTOS 互斥锁
xPCDataMutex = xSemaphoreCreateMutex();

// 访问共享数据前获取锁
if (xSemaphoreTake(xPCDataMutex, portMAX_DELAY) == pdTRUE) {
    // 安全访问 pcData
    xSemaphoreGive(xPCDataMutex);
}

三、总结

MQTT 模块通过 PubSubClient 库实现物联网远程控制，volatile 关键字确保多任务数据一致性。PC 性能监控模块通过多任务架构实现串口接收、数据解析和 UI 刷新，互斥锁保护共享数据安全。
